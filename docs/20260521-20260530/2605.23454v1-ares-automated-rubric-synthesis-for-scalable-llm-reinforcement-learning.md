## 摘要
基于量规的奖励为扩展大语言模型的强化学习（RL）提供了有前景的途径，使其超越具有自动可验证答案的任务。然而，扩展基于量规的强化学习仍然具有挑战性：现有方法通常依赖于专家编写的量规和人工构建的问题集，而固定的任务级量规可能无法捕捉个别问题的评估要求。我们提出 ARES（面向可扩展强化学习的自动评分量规合成），一个大规模自动构建基于量规的强化学习数据的框架。从原始预训练文档开始，ARES 将源知识转换为自包含的问答对，并共同生成特定于问题的加权量规，从而为开放式回答提供实例级奖励监督。为了提高多样性和质量，ARES 在领域标签和角色信息上条件化生成，并应用验证过滤器以确保问题自包含性、答案忠实性和量规有效性。使用 ARES，我们在十个领域构建了 10 万个量规注释的实例。在七个基准上的实验表明，使用 ARES 训练的基于量规的强化学习优于持续预训练、监督微调和二元奖励强化学习，在医疗保健和指令遵循等多维开放式任务上提升最大。

## Abstract
Rubric-based rewards offer a promising way to extend reinforcement learning (RL) for large language models beyond tasks with automatically verifiable answers. However, scaling rubric-based RL remains challenging: existing approaches often rely on expert-written rubrics and manually constructed question sets, while fixed task-level rubrics may fail to capture the evaluation requirements of individual questions. We propose ARES (Automated Rubric synthEsis for Scalable RL), a framework for automatically constructing rubric-based RL data at scale. Starting from raw pretraining documents, ARES converts source knowledge into self-contained question-answer pairs and co-generates question-specific weighted rubrics, enabling instance-level reward supervision for open-ended responses. To improve diversity and quality, ARES conditions generation on domain labels and persona information, and applies validation filters for question self-containment, answer faithfulness, and rubric validity. Using ARES, we construct 100K rubric-annotated instances across ten domains. Experiments on seven benchmarks show that rubric-based RL trained with ARES, outperforms continual pretraining, supervised fine-tuning, and binary-reward RL, with the largest gains on multi-dimensional open-ended tasks such as healthcare and instruction following.

### 1. 论文的核心问题与整体含义

- **研究动机**：当前大语言模型（LLM）的强化学习（RL）训练，特别是从可验证奖励的强化学习（RLVR）方法，在数学、编程等拥有自动可验证答案的领域取得了巨大成功。然而，这种成功暴露了一个根本性局限：奖励信号高度依赖“结果可廉价、可靠地验证”这一前提。
- **核心问题**：
    1.  **领域局限性**：RLVR 主要适用于有自动检查标准答案的任务，无法有效扩展到医疗、写作、指令遵循等复杂、开放式的多维度任务。
    2.  **奖励信号稀疏**：现有的基于结果匹配的奖励通常是稀疏的、二元的（对/错），无法为模型如何满足复杂、多维度的要求提供足够丰富的学习信号。
- **现有方案的不足**：基于评分量规（Rubric）的奖励方法虽有前景（通过多维度标准评估提供更细粒度的反馈），但其规模化面临两大挑战：
    1.  **构建成本高**：通常需要专家针对具体领域手动撰写问题、参考答案和评估标准，难以大规模扩展。
    2.  **量规粒度过粗**：许多量规定义在任务或数据集级别，提供的是通用评估维度，无法针对“每一个具体问题”指定最关键的评估侧重点。
- **整体含义**：本文旨在解决上述瓶颈，通过自动化的方式，为大规模 RL 训练合成特定于问题的细粒度评分量规，从而将 RL 的有效性从封闭的、可验证的领域扩展到广阔的开放式任务中。

### 2. 论文提出的方法论

- **核心思想**：ARES 是一个自动化的数据流水线，其核心思想是将原始的预训练文档直接转化为带有“实例级（instance-level）”加权评分量规的强化学习训练实例，实现从“文档 → (自包含问题, 参考答案, 特定于问题的量规)”的端到端转换。
- **关键技术细节与流程**：
    - **输入与输出**：给定一个原始文档 `d`，流水线输出一个元组 `(q, a*, Rq)`，其中 `q` 是自包含问题，`a*` 是参考答案，`Rq` 是仅针对问题 `q` 的加权评分量规。
    - **奖励计算**：在 RL 训练中，奖励由量规定义：`Rrubric = Σ wk * J(q, y, ck)`。其中，`J` 是一个充当裁判的 LLM，用于判断模型生成的回答 `y` 在多大程度上满足了评估标准 `ck`，`wk` 是该标准的权重（正分奖励优点，负分惩罚缺点）。
    - **ARES 六阶段流水线**：
        1.  **文档过滤**：使用 LLM 分类器剔除低质量、模板化或缺乏足够上下文信息的预训练文档。
        2.  **领域与角色分配**：为保留下来的文档打上语义领域标签（如医学、技术、社会科学等），并分配三个目标用户角色（如医生、学生、护理人员），以增强生成问题的多样性。
        3.  **量规增强的问答联合生成**：这是核心步骤。一个 LLM 生成器 `Mgen` 以文档、领域和角色信息为条件，**一次性联合生成**问题 `q`、参考答案 `a*` 和特定于问题的加权量规 `Rq`。这确保了评估标准与源知识和预期答案高度对齐。
        4.  **质量验证**：检查生成的问题是否自包含（无需看原文即可回答）、答案是否忠实于源文档、问题是否泄露了答案信息。
        5.  **量规验证**：检查每个评估标准是否清晰、量规是否涵盖了问题的关键要求、权重是否有效且足够多样化。未通过验证的实例将被丢弃。
        6.  **格式转换**：将最终通过验证的数据序列化为 GRPO 等 RL 训练框架兼容的格式。

### 3. 实验设计

- **基础模型**：所有实验均基于 `Qwen3-4B-Base` 模型，从头开始进行不同策略的训练。
- **基准方法（Baselines）**：
    1.  **CPT（持续预训练）**：在 ARES 所使用的相同源文档上进行持续预训练。
    2.  **Natural Reasoning**：在 `NaturalReasoning` 数据集上进行监督微调（SFT），作为一个强大的现货SFT基线。
    3.  **Webscale**：遵循 WEBSCALE-RL 方法，对格式化的问答对应用二元奖励的 GRPO 强化学习，以隔离“量规奖励”的效果。
    4.  **ARES-SFT**：在 ARES 生成的问题和参考答案上进行监督微调，以隔离“强化学习”范式的效果。
- **试验方法（ARES-RL）**：使用 GRPO 算法，并采用问题特定的量规奖励进行训练。训练共 3 个 epoch，批次大小为 128。
- **评估基准（Benchmarks）**：覆盖了七个不同类型的基准测试，以全面评估模型能力：
    - **知识与推理**: MMLU-Pro
    - **数学**: GSM8K
    - **代码生成**: HumanEval+, MBPP+
    - **开放式/多维度任务**: HealthBench (医疗), WritingBench (写作), IFEval (指令遵循)
- **对比目标**：通过对比实验，系统性地分离了三个核心因素：(1) 模型是接触纯文本还是问答对；(2) 训练信号是模仿学习（SFT）还是强化学习；(3) 奖励是二元结果验证还是多维度量规评估。

### 8. 不足与局限

- **模型规模验证单一**：所有实验仅在 `Qwen3-4B-Base`（40亿参数）这一种相对较小的模型上进行，未验证方法在更大规模模型（例如 700 亿参数以上）上的效果。大模型可能具备更强的基线推理能力，此时量规奖励的作用和动态可能会有所不同。
- **裁判模型能力的限制**：奖励计算的可靠性高度依赖于裁判 LLM（`Qwen3-32B`）的判断能力。受限于算力，未能使用更强的裁判模型。如果裁判模型在某些复杂标准上的判断有偏差，将会直接影响 RL 训练的信号质量。这种限制在论文中被定为工程问题而非根本性局限。
- **自动生成的潜在偏差风险**：论文在“更广泛影响”部分坦承，自动化生成量规可能编码源文档或合成 LLM 中的偏见，裁判 LLM 也会引入模型依赖性偏差。使用这些数据训练的模型在安全和公平性上存在潜在风险。
- **特定基准表现**：在 MMLU-Pro 这种多项选择题基准上，ARES-RL 略逊于 ARES-SFT，表明量规引导的开放式 RL 训练策略可能并非对所有任务类型都是最优解，尤其是那些更依赖直接记忆和精确匹配的任务。
