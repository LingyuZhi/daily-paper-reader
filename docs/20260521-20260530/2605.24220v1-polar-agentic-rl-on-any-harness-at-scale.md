## 摘要
语言智能体的强化学习越来越依赖于定制化的运行环境，这些环境管理长期上下文、多轮工具使用和多智能体协调。然而，将这些环境移植到强化学习环境接口中仍然困难，且常常丢失重要的训练信号。我们通过Polar弥合这一鸿沟，它是一个面向任意智能体环境的可扩展异步强化学习推演框架。Polar将智能体环境视为黑盒：它代理LLM API调用，记录令牌级别的模型交互，并重建令牌保真的轨迹用于训练。每个推演节点并行高效地管理运行时预热、智能体执行、轨迹重建和评估，提供异步服务端点，可被独立的训练器大规模消费。这种解耦设计使Polar与智能体环境、训练基础设施和强化学习算法无关，同时提高了长时间运行的智能体工作负载的计算利用率。我们通过使用流行的编码环境训练软件工程任务的智能体来验证Polar。利用简单的GRPO，Polar在SWE-Bench Verified基准上，分别使用Codex、Claude Code、Qwen Code和Pi环境，将Qwen3.5-4B的性能提高了22.6、4.8、0.6和6.2个点。我们进一步展示了Polar在自定义环境上的离线数据生成，并消融研究了轨迹重建策略。Polar改写了其前身Prorl Agent，并已注册为NeMo Gym环境之一。

## Abstract
Reinforcement learning for language agents increasingly depends on custom harnesses that manage long-running context, multi-turn tool use and multi-agent orchestration. However, porting these harnesses into RL environment interfaces remains difficult and often loses important training signals. We bridge this gap with polar, a rollout framework for scalable asynchronous RL over arbitrary agent harnesses. Polar treats the agent harness as a black box: it proxies LLM API calls, records token-level model interactions, and reconstructs token-faithful trajectories for training. Each rollout node efficiently manages runtime prewarming, agent execution, trajectory reconstruction, and evaluation in parallel, exposing asynchronous service endpoints that can be consumed by independent trainers at scale. This decoupled design makes Polar agnostic to agent harnesses, training infrastructure, and RL algorithms while improving compute utilization for long-running agent workloads. We validate polar by training agents on software-engineering tasks with popular coding harnesses. Using simple GRPO, polar improves Qwen3.5-4B by 22.6, 4.8, 0.6 and 6.2 points on SWE-Bench Verified with the Codex, Claude Code, Qwen Code and Pi harnesses, respectively. We further demonstrate Polar for offline data generation over custom harnesses and ablate trajectory reconstruction strategies. Polar rewrites its preceding work, Prorl Agent, and has been registered as one of NeMo Gym environments.

### 2. 论文提出的方法论

`Polar` 的核心思想是将集成边界从智能体环境内部转移到所有LLM智能体共有的**模型API边界**。它由两个核心组件构成：一个推演服务器（Rollout Server）和多个网关节点（Gateway Nodes）。

*   **核心思想：基于代理的推演与重建**
    *   **模型API代理（Proxy）**：`Polar` 在智能体环境和推理服务器之间放置一个兼容多种主流API（如Anthropic, OpenAI Chat, Google风格）的代理。该代理无侵入地转发模型调用，并同时记录提示词、采样令牌、对数概率和响应等令牌级数据。
    *   **令牌保真的轨迹重建（Token-Faithful Trajectory Reconstruction）**：`Polar` 将记录的原始模型交互数据重建为训练器可消费的标准轨迹，包含`prompt_ids`, `response_ids`, `loss_mask`等。这确保了训练信号精确地附着在行为策略采样的令牌上，避免了重编码（retokenization）带来的偏差。

*   **关键技术细节：推演即服务（Rollout-as-a-Service）架构**
    *   **异步推演暂存**：每个网关节点将一次完整的智能体会话分解为独立的、并行的阶段：`INIT`（运行时预热）、`READY`（缓冲）、`RUNNING`（代理执行）和`POSTRUN`（轨迹构建、评估与清理）。这种解耦使得CPU密集的运行时准备和长尾评估工作不会阻塞GPU密集的智能体执行。
    *   **轨迹重建策略**：论文提供了两种轨迹构建器：
        *   **`per_request` (保守基线)**：将捕获的每一次模型调用都视为一个独立的训练轨迹，虽然无损，但会将多轮会话碎片化。
        *   **`prefix_merging` (前缀合并)**：通过检查提示词的严格令牌前缀关系，将属于同一连贯对话链（考虑子智能体、上下文压缩等因素后会形成多个链）的多个模型调用合并为更长、更连贯的轨迹。它会精确地**只将行为策略采样的助手令牌标记为可训练（`loss_mask=1`）**，而将环境插入的规范渲染部分（如工具返回结果）遮罩掉。公式表示为：对于同一链中的连续补全 \(C_{i_m}\) 和 \(C_{i_{m+1}\)，其前缀关系满足 \(p_{i_{m+1}}[1 : |p_{i_m}|] = p_{i_m}\)。最终轨迹的令牌序列为 \(z^{(j)} = p_1 || a_1 || u_1 || a_2 || u_2 || ... || a_K\)，其中 \(a\) 为采样的助手令牌，\(u\) 为环境插入的间隙令牌，损失掩码仅在 \(a\) 上为1。

*   **算法流程**：1) 训练器通过`Polar`服务提交任务；2) 推演服务器将任务分解为多个会话，调度到网关节点；3) 网关节点在隔离的运行时（如Docker）中启动并配置智能体环境，其模型API地址指向本网关的代理；4) 智能体在环境中执行，其LLM调用被代理透明捕获；5) 执行完毕后，会话进入`POSTRUN`阶段，网关根据捕获的补全序列和选择的策略（如`prefix_merging`）构建保真轨迹；6) 内置或自定义的评估器在可能的新运行时中对执行结果（如代码补丁）评分，并将奖励附加到轨迹上；7) 最终，完整的训练样本（轨迹+奖励）通过回调或API返回给独立的RL训练器（如Slime）。

### 7. 优点

*   **极强的通用性和非侵入性**：其最大的创新和优点是**将集成边界定义在LLM API上**，使其能够以黑盒方式支持未来任何只要是基于LLM API调用的智能体环境，从根本上解决了RL框架与环境深度耦合的痛点。
*   **精巧的系统架构设计**：`Rollout-as-a-service`和异步节点内暂存的设计，实现了推演和训炼的解耦，优化了异构计算（CPU/GPU）资源的利用，并能灵活扩展。
*   **对训练信号保真度的重视**：`token-faithful` 重建和 `loss_mask` 的精确应用，从原理上保证了RL算法信用分配的准确性，避免了因重编码或上下文污染导致的训练偏差。
*   **实验验证全面且有深度**：不仅验证了多环境的性能提升，还通过消融实验揭示了计算效率的巨大差异，并通过离线数据生成展示了平台的扩展能力，构成了一个坚实且完整的验证闭环。

### 8. 不足与局限

*   **实验对象单一**：论文仅在软件工程（代码）这一领域进行了验证。虽然理论上可以支持任何LLM智能体，但其在网页浏览、操作系统交互等其他领域的有效性尚待证明。
*   **轨迹重建的简化假设**：`prefix_merging` 虽然高效，但其正确性依赖于检查提示词的严格前缀关系。如果环境对提示词进行了非追加式的复杂重构，可能导致合并失败或轨迹链断裂，此时会回退到碎片化的`per_request`策略。
*   **评测器依赖**：`Polar` 本身不提供奖励信号，其训练效果完全依赖于下游的、可被注册的评估器。对于没有明确、可自动化评估标准的开放域任务（如创意写作），其应用会受限。
*   **算力细节不完整**：论文未披露在线RL训练实验的GPU用量和时间，这就很难精确评估其训练成本效益，这在与其他RL框架的比较中是一个信息缺失。
*   **代理协议覆盖风险**：虽然其代理支持多种主流API，但对未来新出现或快速迭代的闭源模型API（如果该API不兼容），可能存在适配滞后的风险。
