## 摘要
为大规模语言模型配备显式技能已成为使自主智能体解决复杂任务的一种有前景的范式。智能体技能本质上可分为用于广泛认知迁移的通用技能和用于动态执行的任务特定技能。然而，现有基于技能的强化学习方法通常强制在完全外部化和完全内部化之间做出僵化选择，前者导致巨大的上下文开销，后者则面临过拟合和知识冲突的风险。为解决这一困境，我们提出Skill0.5，一种新颖的智能体强化学习框架，通过结合通用技能内化与任务特定技能利用来显式区分技能处理方式。Skill0.5由动态、难度感知的路由器驱动，将任务分流到不同的掌握层级，以应用量身定制的优化策略：对于困难任务，通过特权蒸馏内化通用技能以构建认知基础；对于简单任务，则使用诊断探测来惩罚捷径并强制特定技能的利用。在ALFWorld和WebShop上的实验表明，Skill0.5优于基于记忆和基于技能的强化学习基线，在分布内和分布外场景下均实现了性能提升。

## Abstract
Equipping large language models with explicit skills has emerged as a promising paradigm for enabling autonomous agents to solve complex tasks. Agent skills can be inherently divided into general skills for broad cognitive transfer and task-specific skills for dynamic execution. However, existing skill-based reinforcement learning (RL) methods typically force a rigid choice between full externalization, which incurs prohibitive context overhead, and full internalization, which risks overfitting and knowledge conflicts. To address this dilemma, we propose Skill0.5, a novel agentic RL framework that explicitly differentiates skill treatments by combining general skill internalization with task-specific skill utilization. Driven by a dynamic, difficulty-aware router, Skill0.5 streams tasks into distinct mastery tiers to apply tailored optimization strategies: it internalizes general skills via privileged distillation to build a cognitive foundation for hard tasks, while using diagnostic probing on easy tasks to penalize shortcuts and enforce specific skill utilization. Experiments on ALFWorld and WebShop demonstrate that Skill0.5 outperforms both memory-based and skill-based RL baselines, yielding performance improvements across both in-distribution and out-of-distribution scenarios.

## 1. 核心问题与整体含义
- **技能处理的“两难困境”**：为LLM智能体配备显式技能已成为提升复杂任务解决能力的重要范式。技能通常可分为两类：提供跨域认知基础的**通用技能**（如元推理、错误恢复）和绑定于具体任务的**任务特定技能**。现有基于技能的强化学习方法通常采取两种极端策略：
  - **完全外化**：将所有技能作为上下文注入提示，随着技能库扩大，上下文长度爆炸式增长，严重损害推理和指令遵循能力。
  - **完全内部化**：将所有技能永久写入模型参数，面临模型容量限制、过拟合以及新旧技能知识冲突（导致执行幻觉）。
- **研究动机**：实际部署中，任务特定技能会动态更新，且智能体经常遇到未见过的分布外（OOD）任务及相应技能。全外化与全内化均难以应对这种场景。因此，需要一种**差异化处理**方式：**内化通用技能以构建高效的认知基础，同时动态利用即插即用的任务特定技能来增强适应性**。

## 2. 方法论：Skill0.5 框架
- **核心思想**：基于实时任务掌握度（成功率），用一个动态的难度感知路由器将任务划分到三个掌握层级，并对每个层级施加针对性的优化目标，从而联合实现通用技能内化与任务特定技能利用。
- **关键技术细节**：
  - **技能解耦与上下文构建**：
    - 通用技能池 \(S_G\) 和特定技能池 \(S_S\)（通过嵌入相似度检索 Top-K）。
    - 训练阶段根据任务层级定制上下文 \(c_t\)：困难任务使用包含通用技能的`PRIVILEGED PROMPT`；简单任务使用仅含特定技能的`STANDARD PROMPT`或完全移除技能的`NO-SKILL PROMPT`作为诊断探针。
  - **阶段一：难度感知路由**（使用仅含特定技能的标准提示）：
    - 对每个任务采样 \(G\) 条轨迹，计算经验成功率 \(p_i\)。
    - **硬任务**：\(p_i = 0\)。
    - **易任务**：\(p_i > \eta_t\)，其中 \(\eta_t\) 为过去 \(W\) 步批均值的滑动平均。
    - **中等任务**：\(0 < p_i \le \eta_t\)。
  - **阶段二：分层优化**：
    - **硬任务：特权蒸馏内化通用技能**。用包含通用技能的教师轨迹筛选成功样本，让学生（仅见特定技能）通过 token 级 Jensen-Shannon 散度（JSD）模仿教师，迫使通用推理逻辑压入参数。
    - **中等任务：标准 GRPO 强化能力**。直接复用阶段一的轨迹进行组内相对优势优化，提升成功率。
    - **易任务：反捷径利用**。用无技能探针采集诊断轨迹，计算利用增益 \(u_i = p_i - p_i^{\text{none}}\)，并与历史均值对比得到任务级利用优势 \(A_{u_i}\)。将该优势叠加到轨迹级优势上，形成复合优势 \( \hat{A}_i^{(g)} \)，以此惩罚不依赖具体技能的捷径行为。
  - **总体目标**：\( \mathcal{L} = \mathcal{L}_{\text{hard}} + \mathcal{L}_{\text{medium}} + \mathcal{L}_{\text{easy}} \)，且对每一任务三者互斥。

## 6. 主要结论与发现
- Skill0.5 在所有 ID 和 OOD 设定下均显著超越全部基线。相较最佳技能基线 SkillRL，ALFWorld ID 提升 +2.3%，OOD 提升 +13.2%；WebShop ID 提升 +2.1%，OOD 提升 +3.9%。
- 提示基和记忆基方法因上下文噪声或缺乏程序性抽象而性能远逊。
- 训练动态显示，Skill0.5 早期借助特权蒸馏快速打破“零梯度困境”，中后期通过反捷径诊断维持 OOD 稳健增长，而 SkillRL 出现过度拟合，SKILL0 在 OOD 上被压制，SLIM 因过早淘汰通用技能导致行为崩溃。
- 消融证实内化是能力基础，利用是解锁 OOD 泛化峰值的关键，二者缺一不可。
