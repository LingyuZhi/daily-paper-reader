## 摘要
训练大型语言模型作为检索增强推理代理通常将强化学习与从更强模型中蒸馏的监督微调冷启动相结合。然而，这一范式忽视了两个基本因素：子技能之间的依赖结构，以及蒸馏并非能力获取唯一途径的可能性。我们通过Plan来研究这一点，Plan是一种用于多跳检索的结构化代理行为，它在执行任何检索之前将问题分解为有序的子问题，使得每个搜索步骤都能锚定到预先设计的子问题上，而不是在先前检索到的部分相关文档的影响下漂移。然而，在跨越30亿到140亿参数的三个模型家族中，我们发现相同的奖励信号会引发性质不同的强化学习失败模式。这一现象表明，成功的训练不仅取决于奖励设计，还取决于模型特定的可行性条件：足够的初始熵、训练稳定性和必备的子技能。受此启发，我们提出了一种自启动范式，其中一个小规模的种子模型生成经过过滤的轨迹，在任何目标模型中激活Plan，从而无需从外部更强模型进行蒸馏。我们的方法在所有测试模型上都激活了Plan，并在多跳问答基准上持续优于竞争基线。

## Abstract
Training large language models as retrieval-augmented reasoning agents typically combines reinforcement learning with an SFT cold start distilled from a stronger model. However, this paradigm overlooks two fundamental factors: the dependency structure among sub-skills, and the possibility that distillation is not the only route to capability acquisition. We study this through Plan, a structured agentic behavior for multi-hop retrieval that decomposes a question into ordered sub-questions before any retrieval is performed, so that each search step can be anchored to a pre-designed sub-question instead of drifting under the influence of partially relevant documents retrieved earlier. However, across three model families spanning 3B to 14B parameters, we find that an identical reward signal induces qualitatively different RL failure modes. This phenomenon indicates that successful training hinges not only on reward design but also on model-specific feasibility conditions: sufficient initial entropy, training stability, and prerequisite sub-skills. Motivated by this, we propose a self-bootstrapping paradigm in which a small-scale seed model generates filtered trajectories that activate Plan in any target model, eliminating the need for distillation from an external stronger model. Our pipeline activates Plan across every tested model and consistently outperforms competitive baselines on multi-hop QA benchmarks.

### 2. 论文提出的方法论

*   **核心思想：**
    1.  **结构化代理行为：Plan**：在回答多跳问题时，模型首先生成一个全局计划 `<plan>`，将主问题分解为有序的 Step 1, Step 2… 等子问题。随后的每个执行步骤（思考、搜索、文档、精炼）都明确锚定在一个特定的子问题上。
    2.  **自举（Self-Bootstrapping）训练范式**：利用一个能够通过RL成功习得Plan的小型“种子模型”生成高质量轨迹，经过滤后作为SFT数据，激活任何其他目标模型的Plan能力，从而省去从外部超强模型蒸馏的依赖。
*   **关键技术细节与公式：**
    *   **轨迹结构**：`o = (τ0, τ1, ..., τK)`。`τ0` 为全局 `<plan>`；每个 `τk` 由 `<think>`, `<search>`, `<documents>`, `<refine>` 构成，对应处理第k个子问题。轨迹以 `<answer>` 结束。
    *   **计划感知奖励（Plan-Aware Reward）**：
        *   总分由结果奖励（`R_ans`，F1分数）和仅在答案分零时激活的辅助奖励（`R_aux`）构成：`R = R_ans + I[R_ans = 0] · R_aux`。
        *   辅助奖励包括格式奖励（`R_fmt`）和 Plan 奖励（`R_plan`）。
        *   **Plan 奖励**：核心创新。它计算计划中第k步的子问题 `p_k` 与对应思考步骤 `t_k` 之间的语义对齐分数 `S_align`（平均 token 级 F1 分数）。为防止策略抄袭计划内容，使用了阈值化函数：`R_plan = 1` 若 `S_align > δ`（δ=0.25），否则 `R_plan = S_align`。
    *   **自举训练流程**：
        1.  **构建种子模型**：对满足可行性条件的 `Qwen2.5-3B-Base` 进行两阶段RL（先习得精炼，再习得计划）。
        2.  **轨迹生成与过滤**：使用种子模型在训练集生成 Plan 轨迹，并通过**硬过滤**（格式有效、答案正确）和**软评分**（加权组合搜索轮次、查询多样性、精炼密度），最终通过按搜索步数分桶采样获得4500条高质量轨迹。
        3.  **目标模型激活**：对任意目标模型进行SFT（使用上述过滤数据），随后使用计划感知奖励进行RL微调。

### 6. 论文的主要结论与发现

1.  **RL 训练失败的三大条件**：直接 RL 训练 Plan 行为需要同时满足三个模型特定的可行性条件：**足够的初始熵**（避免先验坍缩）、**训练稳定性**（避免后期熵爆炸）、**必备的子技能**（如必须先掌握信息精炼，才能学会计划）。
2.  **异构失败模式**：相同的奖励信号和训练设置在不同模型上会导致性质不同的失败（熵先验坍缩、熵爆炸、无精炼技能导致的波动），这些失败的根源是违反了上述条件。
3.  **自举范式的有效性**：提出了一种有效的自启动范式。一个满足条件的小型种子模型（3B）生成的过滤轨迹，不仅能成功激活任何目标模型的 Plan 能力，**其效果甚至优于从 72B 强模型蒸馏而来的轨迹**。
4.  **蒸馏数据的分布对齐问题**：从强模型（72B）蒸馏的轨迹可能位于目标模型的“可达分布”之外，这会导致 SFT 冷启动后 RL 阶段的不稳定和性能下降。而自举生成的轨迹天然与目标模型分布对齐。

### 7. 优点

*   **深刻的问题洞察**：超越了简单的奖励设计，从子技能依赖和模型可行性条件的角度，深刻分析了 RL 训练复杂代理行为的失败根源。
*   **范式创新**：提出的“自举”训练范式，展示了摆脱对外部超强模型依赖的可能性，且方法在多个模型上被证实更优、更稳定。
*   **方法论严谨**：`Plan` 行为的定义清晰，奖励函数（特别是带阈值的 Plan 奖励）设计精巧，有效防止了奖励黑客（reinforcement hacking）。
*   **实证研究扎实**：在极广的模型范围（3个家族，多种规格）上验证了核心主张，包含全面的基准对比、详尽的定性失败分析和多样的消融实验。

### 8. 不足与局限

*   **任务单一性**：
    *   研究以“Plan”作为一种代表性复合代理行为，其发现的依赖现象是否能推广到其他代理任务（如工具选择、长远规划）仍有待验证。
*   **检索环境静态**：
    *   实验使用静态的 Wikipedia 快照，未在动态、对抗性的真实网络检索环境下测试。这可能影响其在实际应用中的鲁棒性。
*   **种子模型依赖**：
    *   自举范式仍然需要一个能够通过纯RL习得 Plan 的成功“种子模型”。对于种子模型的筛选标准和更广泛的适用性，论文没有提供更深入的分析。
*   **可能的偏差**：虽然论文解释了 `Llama3.2-3B-Base` 上的巨大性能提升是由于其基线模型无法遵循指令而导致，但这在直接对比时仍可能对整体均值造成一定的“噪音”。
