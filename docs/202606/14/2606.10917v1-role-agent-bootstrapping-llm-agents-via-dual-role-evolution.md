## 摘要
尽管大语言模型（LLM）智能体在复杂任务上展现了强劲性能，但其学习常受限于低效的交互反馈与静态训练环境，阻碍了更广泛的泛化。为克服这些局限，本文引入角色智能体（Role-Agent），一个利用单个LLM同时作为智能体与环境运行的框架，实现自举式协同演化。角色智能体包含两个协同组件：世界在智能体（WIA）与智能体在世界（AIW）。在WIA中，LLM充当智能体，并在每次动作后预测未来状态；预测状态与实际状态之间的对齐度随后作为过程奖励，鼓励环境感知推理。在AIW中，LLM分析失败轨迹中的失败模式，并检索具有相似失败模式的任务，从而重塑训练数据分布以实现针对性练习。在多个基准上的实验表明，角色智能体持续提升性能，在强基线上平均获得超过4%的提升。

## Abstract
Although Large Language Model (LLM) agents have demonstrated strong performance on complex tasks, their learning is often limited by inefficient interaction feedback and static training environments, which hinder broader generalization. To address these limitations, this paper introduces Role-Agent, \textcolor{black}{a framework} that harnesses a single LLM to function concurrently as both the agent and the environment, enabling a bootstrapped co-evolution. Role-Agent comprises two synergistic components: World-In-Agent (WIA) and Agent-In-World (AIW). In WIA, the LLM acts as the agent and predicts future states after each action; the alignment between predicted and actual states is then used as a process reward, encouraging environment-aware reasoning. In AIW, the LLM analyzes failure modes from failed trajectories and retrieves tasks with similar failure patterns, thereby reshaping the training data distribution for targeted practice. Experiments on multiple benchmarks show that Role-Agent consistently improves performance, yielding an average gain of over 4\% over strong baselines.

### 1. 核心问题与整体含义（研究动机和背景）

- **问题**：现有LLM智能体（agent）在复杂任务上虽然表现不错，但其学习过程受限于两个关键瓶颈：  
  - **低效的交互反馈**：静态环境提供的反馈稀疏且非特定，无法暴露智能体的隐藏弱点。  
  - **静态训练环境**：环境本身不会根据智能体的能力动态调整，导致智能体难以针对自身缺陷进行针对性练习，泛化能力受限。
- **整体含义**：为了突破这些限制，作者提出让单个LLM同时扮演“智能体”和“环境”两个角色，实现**自举式（bootstrapped）智能体-环境协同演化**。这种双角色设计无需引入额外模型，即可在训练过程中让环境自适应地为智能体提供更精细的反馈和更匹配的训练任务，从而提升智能体的泛化能力和最终性能。

### 2. 方法论：核心思想、关键技术细节、算法流程

- **核心思想**：利用同一LLM轮流扮演智能体和环境，通过两个协同组件实现闭环演化：
  1. **World-In-Agent (WIA，世界在智能体中)**：让LLM在作为智能体执行动作后，预测后续H步的未来状态。通过对比预测状态和实际状态的对齐程度（使用最长匹配子序列LMS计算），生成**预测奖励**，对原始任务奖励进行调制（相乘），从而鼓励智能体进行环境感知的推理。  
  2. **Agent-In-World (AIW，智能体在世界中)**：让同一LLM充当环境角色，分析失败轨迹中的失败模式（如实体混淆、遗漏前置条件等），并检索历史上具有相似失败模式的任务，将它们重新整合到训练数据分布中，从而让智能体能针对自己的历史缺陷进行针对性练习。

- **关键技术细节**：
  - **预测奖励计算**：智能体每步动作后预测未来H步状态，使用LMS计算预测状态与真实状态之间的相似度（0~1），作为预测奖励 `\tilde{r}_{t,h}`。  
  - **奖励调制**：组合任务奖励和预测奖励：`R_t = R_task(at) * (1 + R_pre(at))`。预测奖励仅对非零任务奖励进行缩放，不会为失败轨迹引入额外奖励。  
  - **状态分组与状态级优势**：借鉴GiGPO，将批次中相同状态的动作分组，计算每个状态组内的状态级优势 `A_S`，再与轨迹级优势 `A_E` 加权组合（权重α=1.0），用于策略优化。  
  - **失败模式分析**：对每个失败轨迹，用LLM提取失败类型（如ENTITY_CONFUSION）、触发条件、核心教训和检索查询，存入离线库。  
  - **任务检索与分布重塑**：根据当前失败模式，从离线库中检索相似历史任务，并将它们重新加入训练集，改变采样分布。

- **算法流程**（文字描述）：  
  1. 初始化策略πθ和失败记忆库M为空。  
  2. 每轮训练：采样一批任务 → 对每个任务rollout获得轨迹 → 在每步动作后由同一LLM预测未来H步状态 → 计算预测奖励并调制任务奖励 → 结束rollout后，对批次内相同状态进行分组，计算状态级优势和最终优势 → 用GRPO风格的截断目标更新策略πθ。  
  3. 对每个失败轨迹，用LLM分析失败模式并存储到M中。  
  4. 根据当前失败模式检索M中相似任务，更新任务分布pD。  
  5. 重复直至收敛。
