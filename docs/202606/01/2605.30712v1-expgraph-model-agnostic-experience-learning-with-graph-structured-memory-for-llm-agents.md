## 摘要
大语言模型（LLM）智能体在推理、工具使用和多步交互方面表现出强大的能力，但它们通常从零开始解决任务，无法重用先前经验中的成功策略或失败教训。对收集的经验进行微调可以提高重用性，但当出现更强或更合适的执行器时，这种方法缺乏灵活性。我们提出ExpGraph，一种模型无关的经验学习框架，使得冻结且可替换的LLM执行器能够通过外部经验重用进行改进，而无需参数更新。ExpGraph将历史轨迹总结为可重用的技能和失败教训，将其组织为自进化经验图中的节点，并通过图扩散和效用感知排序检索有用经验。使用强化学习训练一个轻量级检索副驾驶，其反馈来自比较有无检索经验时执行器的性能，同时根据下游任务结果在线更新图。我们在ExpSuite上评估ExpGraph，涵盖问答、数学推理、代码生成以及多步智能体环境（包括ALFWorld和AppWorld）。在静态任务上，ExpGraph分别在使用较小和较大执行器时比最强基线提升12.2%和4.7%；在智能体环境中提升21.4%和12.7%，同时平均交互步骤减少12.7%和21.6%。消融实验表明，图结构经验、效用感知排序和自适应检索共同实现了跨不同任务和执行器模型的有效经验重用。

## Abstract
Large language model (LLM) agents have shown strong capabilities in reasoning, tool use, and multi-step interaction, but they often solve tasks from scratch and fail to reuse successful strategies or failure lessons from prior experience. Fine-tuning on collected experience can improve reuse, but it is inflexible when stronger or more suitable executors emerge. We propose ExpGraph, a model-agnostic experience learning framework that enables frozen and replaceable LLM executors to improve through external experience reuse without parameter updates. ExpGraph summarizes historical trajectories into reusable skills and failure lessons, organizes them as nodes in a self-evolving experience graph, and retrieves useful experiences through graph diffusion and utility-aware ranking. A lightweight retrieval copilot is trained with reinforcement learning using feedback that compares executor performance with and without retrieved experiences, while the graph is updated online from downstream task outcomes. We evaluate ExpGraph on ExpSuite, covering question answering, mathematical reasoning, code generation, and multi-step agentic environments including ALFWorld and AppWorld. ExpGraph improves over the strongest baseline by 12.2% and 4.7% on static tasks with smaller and larger executors, and by 21.4% and 12.7% in agentic environments, while reducing average interaction steps by 12.7% and 21.6%. Ablations show that graph-structured experience, utility-aware ranking, and adaptive retrieval jointly enable effective experience reuse across diverse tasks and executor models.

## 1. 核心问题与研究动机

- **核心问题**：LLM 智能体虽具强大推理与交互能力，但通常从零开始解决每个任务，无法系统复用以往的成功策略或失败教训；而通过微调执行器来积累经验，在 LLM 快速迭代时会因模型更换而需要重复训练，缺乏灵活性。

## 2. 方法论：ExpGraph 框架

### 核心思想
- 将历史轨迹提炼为**技能**（成功）与**教训**（失败）等可重用的经验单元，组织成**自进化的经验图**。
- 采用**检索副驾驶（retrieval copilot）** 根据当前任务自适应控制图扩散广度（R）与相似-效用权衡（W），并通过图扩散和效用感知排序选取经验。
- 使用**强化学习（PPO）** 优化检索策略，奖励设计为“有经验时任务分数”减去“无经验时任务分数”加上绝对分数正则化项。
- 执行器保持**冻结**，经验图与检索副驾驶在线更新，实现模型无关。

### 技术细节
- **经验图构建**：每个节点包含文本描述、嵌入向量、估计效用、检索次数；新节点与 top-K 邻居（余弦相似度 ≥θ）连边。
- **语义种子**：通过余弦相似度选择 top-m 节点作为种子集。
- **图扩散**：基于种子集的个性化 PageRank，重启概率 α(ρ) 由参数 R 控制（ρ = R/100），R 越大扩散越局限。
- **效用感知排序**：候选节点得分 = (1-λ)×语义相似度 + λ×（效用 + UCB 探索项），λ = W/100。
- **奖励函数**：r = (s_with − s_without) + η·s_with，η=1。
- **在线图更新**：节点效用通过指数移动平均更新，新轨迹总结后插入图，容量超限时淘汰低效用低频率节点。

### 算法流程（文字描述）
每个训练迭代：采样任务 → 副驾驶输出 R,W → 语义种子 → 图扩散扩展 → 效用排序选 top-K → 有经验执行 → 无经验执行 → 计算奖励 → PPO 更新副驾驶 → 更新节点效用 → 新轨迹总结插入图 → 淘汰低质量节点。

## 3. 实验设计

### 数据集与场景
- **ExpSuite-Static**：问答（ARC-C、CommonsenseQA、GPQA、MMLU、OBQA）、数学推理（GSM8K、GSM-Symbolic、MATH）、代码生成（HumanEval+、MBPP+），共 10 个基准。
- **ExpSuite-Agentic**：ALFWorld（Seen/Unseen，按成功率与步骤数）、AppWorld（Test-N/Test-C，按通过率与步骤数），共 4 个测试集。
- 每个静态数据集划分 5:2:3 为训练/验证/测试；agentic 使用官方划分。

### 执行器
- 静态任务：小规模 Llama-3.2-3B-Instruct，大规模 Llama-3.1-8B-Instruct。
- 智能体环境：小规模 Qwen3-32B，大规模 Gemini-3.1-Flash-Lite。
- 所有执行器保持冻结，仅通过输入-输出接口使用。

### 对比基线
- 无记忆（No-Memory）
- 检索中心：ReasoningBank、ExpeL、LightMem、Mem0、AWM、MemRL
- LLM 中心：IRCoT、Search-o1、S3
- 提示基线（仅 agentic）：ReAct、Reflexion
- 公平性：所有经验基线与 ExpGraph 使用相同数量的历史轨迹。

### 训练与评估
- 每任务执行两次（有/无经验）以计算奖励。
- 评估时副驾驶与图固定，贪婪解码输出参数。

## 4. 资源与算力

- **硬件**：4 块 NVIDIA A6000 GPU（48GB），BF16 混合精度，FSDP 并行（verl 框架）。
- **时长**：
  - ExpSuite-Agentic（ALFWorld）500 步训练约 40 小时。
  - ExpSuite-Static 300 步训练约 20 小时。
- **其他**：使用 vLLM 加速推理，梯度检查点，经验图服务器运行于 CPU 并通过 HTTP 通信。

## 5. 实验数量与充分性

- **全面性**：覆盖静态 10 个基准×2 个执行器 + agentic 4 个测试集×2 个执行器，共 28 个主要配置。
- **消融实验**：4 个变体（去除相似度过滤、扁平记忆、去除图扩散、去除效用排序），在 5 个领域（QA、推理、代码、ALFWorld、AppWorld）下对比，验证各组件必要性。
- **转移实验**：3 种设定（小→大、大→小、非推理→推理），在 5 个领域下评估 4 种转移方式（仅图、仅副驾驶、图+副驾驶、完整 ExpGraph）。
- **公平性**：所有基线使用相同数量的历史经验；副驾驶与执行器分离训练，避免信息泄漏。
- **充分性**：实验设计合理、对比公平、消融完整，足以支撑主要结论。

## 6. 主要结论与发现

1. **总体最佳性能**：静态任务上比最强基线提升 12.2%（小执行器）和 4.7%（大执行器）；agentic 环境提升 21.4%（小）和 12.7%（大），同时交互步数减少 12.7%~21.6%。
2. **经验重用对弱执行器与长周期任务更有利**：小执行器、agentic 场景增益更大。
3. **强零样本迁移能力**：图+副驾驶组合从弱到强、从非推理到推理执行器迁移接近全量训练效果；从强到弱迁移虽有下降但仍有效。
4. **各组件均有贡献**：图结构最关键（扁平记忆降幅最大），图扩散与效用排序对 agentic 任务尤其重要。

## 7. 优点

- **模型无关**：执行器完全冻结，不依赖其架构、梯度或参数，兼容闭源模型，降低部署成本。
- **图结构组织经验**：超越平面最近邻匹配，通过图扩散发现结构相关的策略/子目标/失败模式。
- **效用感知自适应检索**：副驾驶通过下游反馈学习何时扩散多远、何时信任效用而非相似度，训练可适应任务与执行器。
- **在线自进化**：经验图不断吸收新经验、淘汰低效节点，无需离线重建。
- **强实证结果**：多个基准、多种执行器、消融与迁移实验均表现一致优势。

## 8. 不足与局限

- **实验覆盖有限**：限于静态推理基准与两个 agentic 环境，未评估更长周期任务（如网页浏览、科学发现、多智能体协作）。
- **图超参数固定**：邻居数 K、相似度阈值 θ、图容量上限、效用更新率 β 等为固定值，未来可能需自适应策略。
- **奖励信号粗糙**：仅使用任务最终分数标量，缺少过程级信用分配（如中间步骤奖励）。
- **提示条件的不一致性**：经验通过 prompt 注入执行器，提示指令可能无法稳定转化为预期行为，存在解释不一致风险（如文献[12, 13, 47]指出）。
- **检索副驾驶规模**：使用 Qwen2.5-3B-Instruct，虽然轻量，但在复杂任务上可能仍有推理瓶颈。
