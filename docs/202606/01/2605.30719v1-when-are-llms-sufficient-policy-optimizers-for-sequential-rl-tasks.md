## 摘要
我们研究大型语言模型（LLM）何时可以充当强化学习（RL）任务的有效黑盒策略优化器，即何时可以用LLM替代经典RL算法？我们通过引入提示策略优化（PromptPO）来探索这个问题，该方法使用状态空间、动作空间和奖励函数的Python描述提示LLM，然后让其根据rollout反馈生成并改进可执行策略。在硬探索环境、Meta-World机器人任务以及几个真实世界控制问题中，PromptPO通常与标准RL基线性能相当或更优，同时使用的环境交互次数显著更少。为了最大化期望回报，且无需进一步显式提示，PromptPO输出的策略范围从调谐的比例控制器或基于规则的规划，到运行价值迭代等规划算法的策略。我们的结果表明，当LLM能够利用关于环境或优化策略的先验知识时，基于LLM的策略优化是充分的。PromptPO在MuJoCo领域中的表现不如标准RL基线，这显示了基于LLM的策略优化在需要精细连续控制的场景中可能存在局限性。

## Abstract
We study when large language models (LLMs) can serve as effective black-box policy optimizers for reinforcement learning (RL) tasks, i.e., when can we replace classical RL algorithms with an LLM? We explore this question by introducing Prompted Policy Optimization (PromptPO), an iterative method that prompts an LLM with Python descriptions of the state space, action space, and reward function, then has it generate and refine executable policies based on rollout feedback. Across hard exploration environments, Meta-World robotics tasks, and several real-world control problems, PromptPO often matches or exceeds the performance of standard RL baselines while using substantially fewer environment interactions. To maximize expected return, and without further explicit prompting, the policies PromptPO outputs range from tuned proportional controllers or rule-based plans to policies that run planning algorithms like value iteration. Our results demonstrate that LLM-based policy optimization is sufficient when the LLM can leverage prior knowledge about the environment or optimization strategy. PromptPO underperforms standard RL baselines in MuJoCo domains. This demonstrates possible limitations of LLM-based policy optimization to settings that requiring fine-grained continuous control.

## 1. 核心问题与整体含义
- **研究动机**：经典强化学习（RL）算法通常需要大量环境交互、精细的超参数调优和算法选择，实践中难以部署。大型语言模型（LLM）在大规模预训练中积累了丰富的知识，可能作为黑盒优化器替代传统RL算法，从而简化使用流程、提升样本效率。
- **核心问题**：LLM 能否以及何时能够作为顺序 RL 任务中充分有效的策略优化器？即用 LLM 替代经典 RL 算法，仅通过 Python 描述的状态空间、动作空间和奖励函数，生成并改进策略以最大化期望回报。

## 2. 方法论
- **核心思想**：提出 **Prompted Policy Optimization (PromptPO)**，一种迭代式的 LLM 驱动策略优化方法。LLM 接收环境描述（状态、动作、奖励函数的 Python 代码），生成可执行策略，通过环境 rollout 获得回报，再反馈给 LLM 进行反思和改进。
- **关键技术细节**：
    - **输入**：状态空间、动作空间、奖励函数的 Python 描述（不提供转移动力学，保持无模型设置）。
    - **评估函数生成**：LLM 首先生成一个 `Feedback` 类（含 `summarize_trajectory` 方法），用于从轨迹中提取额外监督信息（如奖励统计量），类似于 REINFORCE 的轨迹级反馈。
    - **策略生成**：每轮采样 N 个候选策略（Python 代码实现），每个策略在环境中 rollout，计算回报并利用生成评估函数评价。
    - **反馈与迭代**：将本轮最佳策略（按回报排序）及其对应的反思（自然语言比较）连同历史最佳策略一并作为上下文，提示 LLM 生成下一轮策略。若出现执行错误，提供错误信息让 LLM 重新实现。
    - **无需额外显式提示**：最终策略可以是比例控制器、规则式计划或运行规划算法（如 Dijkstra、价值迭代），LLM 自动选择策略形式。
- **算法流程**：给定环境描述 → 生成 Feedback 类 → 采样 N 个策略 → rollout 评估 → 选择最佳策略 + 生成反思 → 更新上下文 → 下一轮继续。

## 6. 主要结论与发现
- **LLM 作为策略优化器足够**：在 15/19 个环境中匹配或超越最佳 RL 性能；在 14/19 个环境中更样本高效（其中 11/19 个提升超过一个数量级）。
- **适用条件**：
    - 当 LLM 可以利用先验知识（如环境优化策略、有效控制原理）时表现优异。

    - 在动作空间适合自然语言推理（如末端执行器位移）时表现好；在需要精细连续控制（如关节扭矩的 MuJoCo）时表现差。
