## 摘要
元强化学习通过从相关任务中提取共享结构实现快速适应，但现有的端到端方法通常将任务推断与具身特定控制耦合在一起。这种耦合可能掩盖非参数任务语义，降低样本效率，并限制跨智能体的复用。我们提出了一种元知识复用框架，该框架在动力学简化后的智能体上学习任务级知识，并将其迁移到异构智能体上。该框架使用贝叶斯非参数先验来组织潜在任务模式，并使用高层策略生成任务级幅度引导。为了桥接可复用的任务知识与不同具身形态，我们引入了一个语义-幅度接口和一个轻量级时间适配器，将冻结的元知识转换为时间对齐的子目标，供具身特定的底层控制器使用。在多个运动智能体上的实验表明，与近期最先进的基线相比，我们的框架将最终步跟踪误差减少了94.75%至99.79%，并且在大约使用23.8%的交互数据的情况下实现了可比的部署性能。

## Abstract
Meta-reinforcement learning enables fast adaptation by extracting shared structure from related tasks, but existing end-to-end methods often couple task inference with embodiment-specific control. This coupling can obscure non-parametric task semantics, reduce sample efficiency, and limit cross-agent reuse. We propose a meta-knowledge reutilization framework that learns task-level knowledge on a dynamics-simplified agent and transfers it to heterogeneous agents. The framework uses a Bayesian non-parametric prior to organize latent task modes and a high-level policy to generate task-level magnitude guidance. To bridge reusable task knowledge with different embodiments, we introduce a semantic-magnitude interface and a lightweight temporal adaptor, which convert frozen meta-knowledge into temporally aligned subgoals for embodiment-specific low-level controllers. Experiments on multiple locomotion agents show that our framework reduces final-step tracking error by 94.75% -- 99.79% compared with recent state-of-the-art baselines and achieves comparable deployment performance with about 23.8% of their interaction data.

## 1. 论文的核心问题与整体含义（研究动机和背景）

元强化学习（Meta-RL）旨在通过从相关任务分布中提取共享结构，使智能体能够仅用少量交互快速适应新任务。然而，现有端到端方法存在三个关键缺陷：

- **表示能力不足**：常用固定高斯潜在先验无法刻画非参数、多模态的任务语义结构，导致任务边界模糊。
- **推理与控制耦合**：任务推断与低层控制在同一具身动力学下联合优化，导致元知识编码了智能体特定的运动模式，难以跨形态迁移。
- **样本效率低下**：形成“鸡生蛋”困境——任务推断需要有信息量的轨迹，而有效轨迹依赖于已能执行有意义行为的策略，早期阶段两者均不成熟，尤其对复杂形态智能体。

因此，核心研究问题是：**如何学习可表达的非参数任务级元知识？如何解耦任务推断与具身控制？如何将所学元知识有效复用于不同形态的智能体？**

## 2. 论文提出的方法论

### 核心思想：ReMAP（Reusable Meta-knowledge for Adaptive Policy Transfer）

- **解耦学习**：在动力学简化智能体上独立学习任务推断模块和高层策略，生成任务级幅度引导；目标智能体仅通过低层策略跟踪这些引导信号，不参与任务推理。
- **贝叶斯非参数先验**：使用狄利克雷过程混合模型（DPMM）作为潜在任务空间先验，自动推断任务模式数量，自适应地组织非参数任务聚类。
- **语义-幅度接口（SMAI）**：将冻结元知识模块输出的任务置信度与幅度耦合，生成统一的语义-幅度子目标（gt = mt * bt），供不同智能体的低层控制器解释执行。
- **轻量级时间适配器（步幅预测器）**：在线少量更新，估计每个子目标应执行多少控制步，实现任务级引导与具身动态的时间对齐。

### 关键技术细节

1. **元知识获取（简化智能体上）**：
   - 编码器 q_φ(z|c) 映射上下文窗口 c_t 到潜在变量 z_t（VAE结构），解码器 p_ξ 重构下一状态、奖励和任务置信度。
   - 损失函数包含状态重建损失 L_state、奖励重建损失 L_reward、任务分类损失 L_task 以及软KL正则项（对齐DPMM先验）。
   - 高层策略 π_high 以 (s_t, z_t) 为输入，输出抽象动作 a_high，使用SAC训练；z_t 经 stop-gradient 处理，防止策略梯度污染潜在表示。
   - DPMM 先验通过 Memoized Variational Bayes 在线更新，包含 birth/merge 操作动态调整活动组件数。

2. **解耦低层策略学习（复杂智能体上）**：
   - 使用SAC框架，以 SMAI 信号 g_t 为额外条件输入。
   - 引入课程学习：逐步增加目标幅度、切换频率、轨迹长度和初始状态多样性，使低层策略先学会稳定运动，再学会跟踪变化的幅度命令。

3. **元知识复用（部署阶段）**：
   - 原始状态经 R(i) 映射为抽象状态，构造上下文 → 冻结编码器推断 z_t → 冻结高层策略生成 a_high → 转化为渐进式幅度 m_t → 结合任务置信度生成 SMAI 子目标 g_t → 步幅预测器估计执行步数 k_t → 低层策略执行 k_t 步。
   - 步幅预测器在线最小化执行后误差 L_P。
