## 摘要
在线强化学习广泛用于将大语言模型与奖励信号对齐，但在噪声或错误指定的奖励下训练可能不稳定。我们识别出一种称为方向不一致的失败模式：在批次内，一小部分高奖励滚动产生了与批次大多数强烈不一致的表示空间偏好方向，导致高方差和不稳定的更新。我们提出GEOALIGN，一种用于迭代策略优化中滚动策划的轻量级插件。GEOALIGN (i) 形成提示内偏好对，(ii) 学习每个滚动隐藏状态的在线投影器以集中奖励排序的位移方向，以及(iii) 通过它们与批次共识原型的角度偏差检测方向不一致的滚动，并用提示内稳定的替代方案纠正它们。GEOALIGN仅需前向传播，且额外开销可忽略。在具有学习到的奖励模型的对话对齐和具有二元验证奖励的数学推理中，GEOALIGN提高了最终性能并减少了训练振荡，优于PF-PPO、PAR、PODS和Seed-GRPO。这些结果表明潜在方向共识可作为在线LLM RL的有效可靠性信号。

## Abstract
Online reinforcement learning is widely used to align large language models (LLMs) with reward signals, yet training can be unstable under noisy or misspecified rewards. We identify a failure mode we call directional inconsistency: within a batch, a small set of high-reward rollouts induces representation-space preference directions that sharply disagree with the batch majority, resulting in high-variance and destabilizing updates. We propose geoalign, a lightweight plug-in for rollout curation in iterative policy optimization. Geoalign (i) forms within-prompt preference pairs, (ii) learns an online projector on per-rollout hidden states to concentrate reward-ordered displacement directions, and (iii) detects directionally inconsistent rollouts via their angular deviation from a batch consensus prototype and rectifies them with within-prompt stable alternatives. Geoalign is forward-pass only and adds negligible overhead. Across dialogue alignment with a learned reward model and mathematical reasoning with binary verified rewards, Geoalign improves final performance and reduces training oscillation, outperforming PF-PPO, PAR, PODS, and Seed-GRPO. These results suggest latent directional consensus as an effective reliability signal for online LLM RL.

### 论文核心问题与整体含义（研究动机和背景）

- 在线强化学习（RL）广泛用于将大语言模型（LLM）与人类偏好或任务奖励对齐，但在噪声、错误指定或可被利用的奖励（如奖励模型伪影、奖励黑客）下，训练经常出现不稳定现象：学习曲线振荡、过早平台期、甚至更新后性能倒退。
- 已有稳定性方法主要将奖励视为一个标量可靠性信号，通过裁剪、重新加权或过滤来控制每个 rollout 的贡献，但忽略了**矢量级的学习信号**：两个奖励相似的 rollout 可能在高维空间中指向截然不同的更新方向。
- 论文提出一种新的失败模式——**方向不一致（directional inconsistency）**：在同一个 RL 迭代中，少量高奖励 rollout 的偏好隐含更新方向与批次中大多数 rollout 的方向严重冲突（角度离群），即便它们奖励很高，也会放大更新方差、导致训练振荡。
- 因此，本文旨在利用隐空间中的方向一致性作为一种轻量级的可靠性信号，在线（on-the-fly）检测并纠正严重的方向离群 rollout，从而稳定 LLM 的在线 RL 训练。

### 论文提出的方法论：核心思想、关键技术细节

#### 核心思想
- 基于 rollout 的最后一层隐层表征（detached），在 prompt 内构造偏好对（低奖励→高奖励），得到位移向量（displacement vector）。这些向量在高维空间中分布杂乱（受语言风格等无关因素干扰）。
- 学习一个轻量投影器（3层 MLP）将位移方向映射到奖励更集中、更可分离的低维空间。
- 在投影空间中构建批次级共识原型向量（reward-margin-weighted centroid），计算每个 preference pair 与原型之间的余弦偏差。
- 对每个 rollout，累积其参与的所有 preference pair 的偏差，得到**几何偏差指数（GDI）**。
- 基于 GDI 的核密度估计（KDE），自适应识别严重离群 rollout，并使用同 prompt 内方向一致的高奖励 rollout 替换它们，从而保持 prompt 内 rollout 数量不变，避免改变归一化分母。

#### 关键技术细节
1. **偏好对构造**：对每个 prompt 内的所有严格排序对（ri,k > ri,ℓ）形成集合 P，计算原始位移方向 ui,k,ℓ = norm(hi,k - hi,ℓ)。
2. **投影器训练**：对当前批次的每个方向对（u, -u）训练投影器 Mψ 和一个临时线性 probe w，使用 logistic 损失区分正负方向。每迭代只训练少量步（50步），丢弃 probe，保留 Mψ。
3. **投影后方向**：vi,k,ℓ = norm(Mψ(ui,k,ℓ))。
4. **共识原型**：vproto = norm( Σ (ri,k - ri,ℓ)·vi,k,ℓ )，奖励差加权保证大差异对共识影响更大。
5. **GDI 计算**：对每个 preference pair，si,k,ℓ = 1 - cos(vi,k,ℓ, vproto)；对 rollout yi,k，GDI(i,k) = Σ_{(a,b)∈I(i,k)} si,a,b，其中 I(i,k) 是涉及该 rollout 的所有偏好对索引。
6. **异常检测**：用 KDE 估计 GDI 的 pdf，若某个 rollout 的 pdf 值低于 α·max(pdf)，则标记为异常（α 是敏感度参数，默认 Math 0.05，HH-RLHF 0.12）。
7. **修正**：对每个异常 rollout，从同 prompt 的方向一致 rollout 中按“Best-Random”策略采样一个替代（50% 选择奖励最高的，50% 从剩余均匀采样），替换到经验缓冲中。

#### 算法流程简要描述（Algorithm 1）
- 每轮迭代：收集 batch → 构造偏好对 → 更新投影器 Mψ → 计算投影后方向 → 计算共识原型 → 计算 GDI → 基于 KDE 密度阈值识别异常 → 用同 prompt 稳定 rollout 替换异常 → 用修正后的 batch 执行标准 GRPO 更新。

#### 基础模型与框架
- 模型：**Qwen3-1.7B** 和 **Qwen3-4B**（Instruct 版用于 Math，Base 版先做 SFT 再 RL 用于 RLHF）。
- RL 算法：**GRPO**（Group Relative Policy Optimization），利用组内归一化优势。
- 框架：Trinity-RFT，GPU 节点：8×NVIDIA A100 (80GB)，使用 FSDP 分布式训练，vLLM 加速生成。

#### 对比方法（基线）
1. **BASE-GRPO**：标准 GRPO，无额外处理。
2. **PF-PPO (BR/BW)**：策略过滤，选取最高奖励 vs 随机（BR）或 vs 最低奖励（BW）的 rollout 进行更新。
3. **PAR**：奖励重塑，使用 sigmoid 中心化防止奖励黑客。
4. **PODS**：下采样 rollout，最大化批次内奖励方差。
5. **SEED-GRPO**：基于语义熵的不确定性感知更新。

- Math：多个基准上的平均准确率（取最后 5 个检查点平均）。

### 论文的主要结论与发现

1. **方向不一致是 LLM 在线 RL 中一个关键的不稳定来源**：即使奖励合理，少量 rollout 的方向离群会显著增加更新方差。
2. **GEOALIGN 能有效识别和纠正方向离群 rollout**：仅用前向隐层状态和轻量投影器，即可获得稳定的共识方向，并选择同 prompt 内一致的 rollout 替换。
3. **在数学推理（二进制奖励）和对话对齐（连续奖励）上均显著提升最终性能和训练稳定性**，超越所有对比的鲁棒 RL 基线。
4. **在人为注入奖励噪声时，GEOALIGN 保持更强的鲁棒性**，其 GDI 指标与已知噪声高度相关。
5. **方向投影是集体共识检测的关键**：原始方向散乱，投影后 90% 的向量与共识原型的余弦相似度 > 0.8。

### 优点

- **轻量且即插即用**：只需前向传播，无需额外反向梯度，额外开销（内存 < 500MB，时间 +1%），可作为插件集成到任何策略梯度算法（如 GRPO）中。
- **几何信号而非标量**：利用表示空间中的方向一致性，绕过传统依赖标量奖励加权的局限性，能检测到“高奖励但误导性”的 rollout。
- **保留梯度密度**：通过同 prompt 替换而非丢弃，维持每个 prompt 的 rollout 数量，不改变优势归一化分母，保持有效学习信号。
- **充分消融验证**：对各个设计组件（投影器、密度阈值、修正策略、归一化方式）均进行了系统消融，验证合理性。
- **稳定性与多样性兼得**：通过 Pass@k 和 KL 分析表明，GEOALIGN 降低更新方差的同时不压缩有效解多样性。

### 不足与局限

- **超参数敏感性**：密度阈值 α 需要根据任务提前调整（Math 0.05 vs. RLHF 0.12），论文未提供自适应策略，可能限制迁移到新任务。
- **投影器训练基于当前批次**：每迭代只训练 50 步，可能无法捕获跨迭代的长期方向变化，尤其在奖励分布剧烈漂移时。
- **理论分析局限**：将方向不一致与梯度方差增大的联系限于直观讨论和初步推论，缺少严格理论证明或上界分析。
- **实验范围局限**：仅测试了两种奖励类型（连续与二进制），未扩展到多步过程奖励或更复杂的环境（如代码生成、游戏控制）；模型仅限 Qwen3-1.7B/4B，未验证更大模型或不同架构。
- **计算成本权衡**：相比于纯标量方法（如 PF-PPO），GEOALIGN 需要额外的投影器训练步骤（尽管轻量），且在 batch 较大时构造所有偏好对可能 O(NK²)（论文中 K≤16 可忽略，但若 K 更大需考虑子采样）。未与需要梯度的方向感知方法（如 influence function）进行直接算力比较。
- **潜在过度正则化风险**：虽然 Pass@k 显示多样性保持，但极端情况下过度强调方向一致可能惩罚新颖但稍偏离主流的有益探索路径（论文在 α 过大时观察到性能下降，self 承认这一点）。
