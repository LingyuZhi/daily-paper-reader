## 摘要
基于结果的强化学习为语言智能体提供了稳定的优化基础，但其稀疏的轨迹级奖励对于哪些中间决策应该被强化或抑制缺乏指导。在线策略自蒸馏提供了密集的令牌级监督，但现有的技能条件化变体通常依赖外部技能记忆或检索到的特权上下文，这些方法维护成本高，且可能在与当前策略在多轮交互中产生的状态分布不匹配。我们提出OPID（On-Policy Skill Distillation），一个直接从完整的在线策略轨迹中提取技能监督的框架。OPID将轨迹事后认知表示为分层技能：回合级技能捕捉全局工作流程或失败规避规则，而步骤级技能在关键时间步捕捉局部决策知识。一种关键优先路由机制在识别到关键决策时使用步骤级技能，否则回退到回合级技能作为默认指导。所选技能被注入交互历史中，使得旧策略能够在原始和技能增强的两种上下文中对相同的采样响应重新评分。由此产生的对数概率偏移形成令牌级自蒸馏优势，与结果优势结合用于策略优化。因此，OPID在保持RL作为主要训练目标的同时，引入了密集且分布匹配的事后监督。在ALFWorld、WebShop和基于搜索的QA上的实验表明，OPID在智能体性能、样本效率和鲁棒性上普遍优于纯结果RL和现有技能蒸馏基线。我们的代码可在https://github.com/jinyangwu/OPID/tree/main获取。

## Abstract
Outcome-based reinforcement learning provides a stable optimization backbone for language agents, but its sparse trajectory-level rewards provide little guidance on which intermediate decisions should be reinforced or suppressed. On-policy self-distillation offers dense token-level supervision, yet existing skill-conditioned variants often rely on external skill memories or retrieved privileged context, which are costly to maintain and can be mismatched with the state distribution induced by the current policy in multi-turn interaction. We propose \textbf{OPID} (\textbf{O}n-\textbf{P}olicy Sk\textbf{i}ll \textbf{D}istillation), a framework that extracts skill supervision directly from completed on-policy trajectories. OPID represents trajectory hindsight as hierarchical skills: episode-level skills capture global workflows or failure-avoidance rules, while step-level skills capture local decision knowledge at critical timesteps. A critical-first routing mechanism uses step-level skills when critical decisions are identified and falls back to episode-level skills as default guidance otherwise. The selected skill is injected into the interaction history, allowing the old policy to re-score the same sampled response under both original and skill-augmented contexts. The resulting log-probability shift yields a token-level self-distillation advantage, which is combined with the outcome advantage for policy optimization. OPID thus preserves RL as the primary training objective while introducing dense, distribution-matched hindsight supervision. Experiments on ALFWorld, WebShop and Search-based QA demonstrate that OPID generally improves agent performance, sample efficiency, and robustness over outcome-only RL and existing skill-distillation baselines. Our code is available at https://github.com/jinyangwu/OPID/tree/main.

- **问题**：基于结果的强化学习（如GRPO）为语言智能体提供了稳定的优化框架，但其奖励是稀疏的、轨迹级别的，无法对中间决策进行细粒度信用分配。例如，一个终端奖励只指示成败，却不揭示哪些中间步骤导致了最终结果。

- **本文动机**：提出直接从**当前策略自身完成的在线轨迹**中提取分层技能（回合级和步骤级），通过关键优先路由注入历史，生成**分布匹配**的令牌级自蒸馏优势，与结果优势结合优化策略，无需外部技能库或推理时检索。

## 2. 方法论：核心思想、关键技术细节与算法流程
### 2.1 核心思想
- 将智能体完成的在线策略轨迹视为**事后知识来源**，从中提取**分层自然语言技能**：回合级技能（全局工作流或失败规避规则）和步骤级技能（关键时间步上的局部决策指导）。
- 通过**关键优先路由**，在关键步骤使用步骤级技能，其他步骤使用回合级技能。将路由后的技能注入交互历史，让旧策略对同一采样响应在原始和技能增强两种上下文中重新评分，利用对数概率偏移作为令牌级**技能优势**，与GRPO式的结果优势线性组合，构成最终优势。

### 2.2 关键技术细节
- **在线策略技能提取**：对每个完成的轨迹τ，使用LLM分析器A生成回合级技能 `s_ep(τ)` 和一组步骤级技能 `s_step(τ,t)`（t属于关键时间步集合C_τ）。回合级技能描述成功工作流或失败避免规则；步骤级技能针对关键决策点（如避免重复无效动作、选择下一个检查对象等），简洁、可执行。
- **关键优先路由**：定义路由规则：若t ∈ C_τ，则s(τ,t) = s_step(τ,t)；否则s(τ,t) = s_ep(τ)。即优先使用最具体的可用技能，不盲目叠加。
- **技能条件化自蒸馏**：对每个令牌位置ℓ，原始上下文h(τ,t)下旧策略对数概率为 ℓ_old = log π_θold (y_τ,t,ℓ | h, y_<ℓ)；技能增强上下文˜h下为 ℓ_skill = log π_θold (y_τ,t,ℓ | ˜h, y_<ℓ)。技能优势 A^skill_τ,t,ℓ = (ℓ_skill - ℓ_old) * 掩码。正值表示技能使令牌概率上升，负值表示不一致。
- **策略优化**：最终优势 A^OPID = A^ep + λ_skill * A^skill。A^ep 采用GRPO式的组内轨迹奖励归一化（(R(τ)-μ)/σ）。使用裁剪PPO损失 (`min(r*Adv, clip(r,1-ε,1+ε)*Adv)`) 加KL正则项。

### 2.3 算法流程（文字说明）
1. 采样：对每个任务提示，从当前策略π_θold采样N条在线轨迹。
2. 计算回合优势：在组内归一化轨迹奖励。
3. 技能提取：用分析器从每条轨迹提取回合级和步骤级技能。
4. 路由与评分：对每条轨迹的每个步骤，路由适当技能，在两种上下文中评分同一响应，得到令牌级技能优势。
5. 组合优势：结合回合优势与技能优势。
6. 策略更新：用裁剪PPO优化，保持KL正则。
7. 推理时无需分析器、技能检索或特权上下文。

## 7. 优点
1. **无需外部技能库**：技能直接从当前策略的完整轨迹提取，天然分布匹配，避免了检索不匹配问题。
2. **层级技能+关键优先路由**：回合级技能提供全局稳定指导，步骤级技能提供关键点精确指导；路由机制避免盲目叠加，选择最合适的粒度。
3. **保持RL为主目标**：结果奖励仍为主导信号，技能蒸馏仅作为辅助令牌级塑造信号，训练目标稳定，推理零开销。
4. **实验全面深入**：覆盖多个模型尺度、三个领域、多种基线，包含样本效率、跨域泛化、消融分析，验证了方法的鲁棒性和泛化性。
