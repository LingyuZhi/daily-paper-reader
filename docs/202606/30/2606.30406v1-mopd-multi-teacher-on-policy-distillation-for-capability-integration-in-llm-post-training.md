## 摘要
现代大语言模型在后训练阶段依赖强化学习来提升特定能力，但将多种能力整合到单一模型中仍然困难。现有方法（如离策略微调和混合强化学习）要么效率低下，要么性能下降。本文提出多教师同策略蒸馏（MOPD），一种结合多个领域强化学习教师能力的后训练范式：首先对每个领域进行专门强化学习以获得一组领域教师，然后在学生自身的轨迹上将这些教师蒸馏到学生中。这消除了暴露偏差并提供了密集的优化信号。在Qwen3-30B-A3B上，MOPD优于混合强化学习、级联强化学习、离策略微调和参数合并基线，几乎继承了每位教师的全部能力。MOPD还支持领域教师的并行独立开发，消除了多领域后训练中常见的跨领域耦合。MOPD已部署在工业级前沿模型MiMo-V2-Flash的后训练中，展示了其在容量整合方面的实用价值。

## Abstract
Modern large language models (LLMs) rely on reinforcement learning during post-training to push specific capabilities, yet integrating multiple capabilities into one model remains hard. Existing methods, such as Off-Policy Finetune and Mix-RL, are either inefficient or lose performance. In this work, we propose Multi-teacher On-Policy Distillation (MOPD), a post-training paradigm for combining the capabilities of multiple domain RL teachers: we first run per-domain specialised RL to obtain a set of domain teachers, then distill these teachers into the student on its own rollouts. This eliminates exposure bias and provides a dense optimization signal. On Qwen3-30B-A3B, MOPD outperforms Mix-RL, Cascade RL, Off-Policy Finetune, and Param-Merge baselines, inheriting nearly all of each teacher's capability. MOPD also enables parallel, independent development of domain teachers, removing the cross-domain coupling typical of multi-domain post-training. MOPD has been deployed in the post-training of MiMo-V2-Flash, an industrial-scale frontier model, demonstrating its practical value for capability integration in frontier-scale LLMs.

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**：大语言模型（LLM）后训练中，不同任务领域（数学、指令遵循、软件工程等）各自依赖不同的强化学习（RL）流水线来提升专业能力，但将这些能力高效、稳定地整合到单一模型中是一个开放难题。
- **现有方法的不足**：
  - **混合RL（Mix-RL）**：将多领域提示混合训练，但跨领域信号干扰，产生“跷跷板效应”，联合模型性能低于各领域教师。
  - **级联RL（Cascade RL）**：依次训练各领域，早期能力随后期训练退化，且整体运行长度增加训练不稳定风险。
  - **离策略微调（Off-Policy Finetune）**：用教师生成的固定数据集做SFT，存在暴露偏差（训练与推理分布不匹配）。
  - **参数合并（Param-Merge）**：权重空间融合，不稳定且难以匹配所有教师全部能力。
- **本文目标**：提出一种新的后训练范式MOPD，在策略空间直接进行多教师能力集成，同时满足密集优化、同策略训练、并行化流水线三个实用要求。

## 2. 方法论
### 核心思想
MOPD 采用三阶段流水线：
1. **阶段1：通用SFT**：在覆盖所有目标领域的大规模语料上对基座模型进行微调，得到共享初始化的SFT检查点。
2. **阶段2：领域专门RL训练**：从SFT检查点出发，对每个领域独立、并行地进行RL训练，得到一组冻结的领域专家教师 $\pi_{\phi_d}$。各领域可使用最适合自身的RL配方（如可验证答案RL、沙盒代理RL等）。
3. **阶段3：多教师同策略蒸馏（MOPD）**：学生模型从SFT检查点初始化，教师组冻结。训练时，学生对多领域提示采样生成轨迹，每个轨迹根据任务领域分发给对应教师，教师预填充得到每token的对数概率，然后最小化学生与教师之间的逐token反向KL散度。

### 关键技术细节
- **损失函数**：基于逐token反向KL：
  $$\mathcal{L}_{\text{rev-KL}} = \mathbb{E}_{x, y \sim \pi_\theta} \left[ \frac{1}{|y|} \sum_t \sum_v \pi_\theta(v) \log \frac{\pi_\theta(v)}{\pi_{\phi_d}(v)} \right]$$
- **策略梯度实现**（对应公式(4)）：将教师-学生对数概率差作为逐token优势 $\hat{A}_t = \text{sg}[\log \pi_{\phi_d}(y_t) - \log \pi_\theta(y_t)]$，经双向裁剪后使用标准PPO/GRPO框架优化：
  $$\mathcal{L}_{\text{MOPD}}^{\text{PG}} = -\mathbb{E}_{x, y} \left[ \frac{1}{|y|} \sum_t \text{clip}(\hat{A}_t, -A_{\max}, +A_{\max}) \log \pi_\theta(y_t) \right]$$
- **Top-𝑘蒸馏实现**（对应公式(5)）：仅对教师Top-𝑘 tokens计算反向KL，并加入额外项 $\pi_{\phi_d}(v) - \pi_\theta(v)$ 以纠正截断偏差，保证在Top-𝑘集合上 $\pi_\theta = \pi_{\phi_d}$ 为最小点。该方法降低基础设施负担（传输轻量级top-𝑘 logits而非全词汇分布）。
- **基础设施**：每个领域教师部署为独立异步prefill服务，学生采样与教师prefill重叠，几乎不增加额外wall-clock开销。

### 算法流程（文字说明）
1. 从多领域数据集中采样一批提示。
2. 学生为每个提示生成完整轨迹，并记录每token概率分布。
3. 根据任务领域将轨迹路由到对应教师，教师预填充得到每token对数概率（或top-𝑘 logits）。
4. 使用上述损失函数更新学生参数。
