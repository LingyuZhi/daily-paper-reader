## 摘要
深思熟虑需要时间。在实时环境中，这段时间并非免费。标准强化学习（RL）回避了这一问题，因为环境会无限期等待智能体的决策。相反，我们研究了实时RL环境，其中在等待智能体行动时环境会持续进展。基于先前的实时形式化，我们引入了可变延迟实时强化学习，智能体在每个决策点选择思考多长时间，因为环境在向前推进。对于我们使用的规划智能体，正确的延迟是状态依赖的，而天真地规划“计划多久”可能会使智能体瘫痪。我们转而通过训练一个轻量级的门控策略，置于规划器之上，来选择状态依赖的规划预算。在实时版吃豆人、俄罗斯方块、贪吃蛇、极速六角棋和极速围棋中，我们的门控策略优于固定预算和启发式基线，并且能够迁移到环境和智能体在两个不同GPU上运行的实时设置中。

## Abstract
Deliberating takes time. In real-time settings, that time is not free. Standard reinforcement learning (RL) sidesteps this as the environment waits indefinitely for the agent's decision. Instead, we study real-time RL environments where the environment progresses while waiting for the agent's action. Building on prior real-time formalizations, we introduce variable-delay real-time RL, where the agent chooses how long to deliberate at each decision point since the environment progresses. For the planning agents we use, the right delay is state-dependent, and naively planning how long to plan can paralyze the agent. We instead approach this setting by training a lightweight gating policy on top of a planner to select state-dependent planning budgets. Across real-time Pac-Man, Tetris, Snake, Speed Hex, and Speed Go, our gating policy outperforms fixed-budget and heuristic baselines, and transfers to a real-time setup where the environment and agent run on two different GPUs.

# 详细中文总结

## 1. 核心问题与整体含义（研究动机和背景）

- **标准 RL 的假设缺陷**：传统强化学习（RL）假设环境会无限期等待智能体决策，这与现实世界中的实时交互不符。在实时系统中，环境会持续演化，而智能体的计算时间会直接导致环境状态的变化，使得“完美但延迟的行动”可能远不如“及时但次优的行动”。
- **现有工作的局限**：已有实时 RL 研究（如 Ramstedt & Pal 2019）通常固定决策延迟为恒定步数，或由系统强制指定延迟，没有考虑智能体可以根据状态主动调节思考时间。
- **论文核心问题**：如何在实时 RL 中让智能体 **自动决定每个决策点“思考多久”** （即规划预算），同时避免“规划如何规划”的元推理悖论（meta-reasoning paradox）。

## 2. 方法论：核心思想、关键技术细节、公式与算法流程

### 2.1 核心思想
- **可变延迟实时 RL（Variable-Delay Real-Time RL）**：将智能体视为一个“随时间运行的过程”，而非瞬时函数。环境每帧固定推进，智能体必须在每帧提交动作（否则使用默认动作），而计算时间直接作为环境进展的代价。
- **预解决“规划如何规划”悖论**：通过训练一个极轻量级的门控策略（Gating Policy）来决定每次调用的规划预算（即 MCTS 模拟次数），门控策略的单次前向传播成本远低于一次 MCTS 搜索，从而避免陷入无限递归。

### 2.2 关键技术细节
1. **框架构成**：
   - **反射策略（Reflex Policy）**：一个极快的基准策略（本文使用规划器本身的策略网络，一次前向传播约 2 ms），用于在规划期间提供每帧的占位动作。
   - **即时动作优化计算（Anytime Action-Refinement Computations）**：一组离散的 MCTS 规划预算 `{c_k}`，其中 `k` 表示运行的帧数（模拟次数）。更长的 `k` 产生更好的行动但消耗更多真实时间。
   - **预算化选项（Budgeted Options）**：每个选项 `o_k` 将 `k` 帧的规划过程封装为单一时间扩展动作：前 `k-1` 帧使用反射策略连续控制环境，第 `k` 帧输出 MCTS 选出的最终动作。
2. **元层级半马尔可夫决策过程（SMDP）**：
   - 门控策略 `π_gate(k|s_t)` 在元决策点选择选项 `o_k`，环境执行 `k` 帧后到达 `s_{t+k}`。
   - 元 Bellman 方程：`V(s_t) = E_k[ Σ_{j=0}^{k-1} γ^j r_{t+j} + γ^k V(s_{t+k}) ]`。
   - 使用 **变长广义优势估计（Variable-Duration GAE）** 修正折扣因子 `γ^k` 以适应元步骤的不等长。
3. **门控策略架构**：
   - 输入包括：原始游戏观测、规划器冻结主干提取的空间特征（128维）、规划器的标量价值估计。
   - 网络组成：空间分支（1x1 卷积 + 3 个残差块） → 时间/时钟嵌入 → 融合（拼接） → 策略头（输出各预算 logits）和价值头（输出基线）。
4. **训练流程**：两阶段训练。（1）先用 AlphaZero 方法训练各环境的冻结基础规划器。（2）然后用 PPO 训练门控策略，元决策的奖励是 `k` 帧内累积折扣奖励，使用上述变长 GAE。

## 3. 实验设计

### 3.1 使用场景/环境
- **承诺动作环境（Committed-action）**：规划时环境持续进展，反射策略在中间帧执行动作。
  - Pac-Man（Jumanji 版本）
  - 实时 Tetris（修改版，每步为一次重力滴）
  - Snake（Jumanji 版本）
- **时钟环境（Clock）**：棋盘静态，但每方有思考时钟，规划消耗时钟滴答。
  - Speed Hex（11×11，pgx 库）
  - Speed Go（9×9，pgx 库）

### 3.2 基准对比方法
- **固定预算策略**：对所有 `k ∈ K` 的 always-k 策略（Pac-Man/Tetris/Snake 中 K={1,2,3,4}；Speed Hex 中 K={2,8,32,128}；Speed Go 中 K={16,32,64,96}）。
- **随机策略**：均匀随机选择预算。
- **启发式策略（仅用于时钟环境）**：贪心（总是最大预算）、中峰（midpeak，根据游戏进程分配预算）。
- **额外基线**：策略仅（Policy-only，即只使用反射策略，无 MCTS）。

### 3.3 评估指标
- 承诺动作环境：每轮平均回报（Episode Return）。
- 时钟环境：预期得分（Expected Score，通过选手间比赛得到），并在多个共享时钟预算（T∈{300,1200,2300,3500,4100}）上取平均。

### 3.4 真实部署实验
- 架构：双 GPU（GPU0 运行环境+反射策略，GPU1 运行 MCTS），异步循环。
- 配置：3 种 GPU（H100, A100, A40）× 3 个承诺动作环境 × 5 个帧率（8–12 FPS），共 45 组测量。

## 6. 主要结论与发现

- **门控策略显著优于所有固定预算基线**：在所有 5 个环境中，自适应分配规划预算比任何单一固定预算策略取得更高回报或得分（Pac-Man +10.3%，Tetris +65%，Snake +10.9%，Speed Hex 和 Speed Go 也明显更优）。
- **门控策略表现出合理的状态依赖行为**：在状态危险/复杂时分配更多计算（如鬼魂远、棋盘密集、蛇身长时）；在时钟宽裕时更愿意深度搜索。
- **训练至真实部署迁移成功**：仿真训练的门控策略直接部署到双 GPU 异步系统，在 H100 和 A100 上表现匹配仿真，即使在 A40 最紧张帧率下性能下降也主要在可预测的 deadline 违规机制内。
- **规划质量与推理延迟随模拟次数共同增长**：验证了自适应分配的必要性——固定预算会牺牲关键状态的质量或浪费简单状态的时间。

## 7. 优点

1. **框架创新性**：首次将元推理（门控策略）与实时 RL 的互动协议结合，提出可变延迟形式化，避免了“规划如何规划”的悖论。
2. **方法轻量且可复用**：门控策略不改变内部规划器，只需冻结基础规划器并训练极小的附加网络，易于替换不同规划器。
3. **仿真到真实迁移验证**：设计了与实际异步执行一致的训练协议（承诺动作协议），证明了直接迁移的有效性，具有实际部署潜力。
4. **实验充分且可视化清晰**：提供丰富的状态-预算关联分析和输入消融，深入解释门控策略为何有效。
5. **代码开源**：提供了完整代码和检查点，可重复性高。

## 8. 不足与局限

1. **依赖完美环境模拟器**：MCTS 内部假设完全已知的环境动力学（AlphaZero 范式），不能直接用于需要学习环境模型的问题（如 MuZero）。作者指出延伸至 MuZero 是未来工作。
2. **冻结基础规划器的限制**：门控策略与基础规划器解耦，附录 F 表明基础规划器选择对下游性能影响大（如 Snack 必须用 k=3 训练的基础模型）。联合优化规划器和门控可能更好。
3. **离散的、手工校准的预算集合**：每个环境的预算集 `K` 是手工选定的固定离散值，没有考虑连续或层级化的计算分配，扩展性受限于预算数量。
4. **训练成本较高**：两阶段训练需要先训练基础规划器，再训练门控策略，且门控策略的 rollout 需要为所有预算并行计算（虽经优化但仍额外开销）。对大规模问题可能成本较高。
5. **实验环境种类有限**：虽然包含 5 个游戏，但都属于棋盘游戏或简单控制任务，未在机器人控制、自动驾驶等高维连续实时任务中验证。迁移实验中仅测试了承诺动作环境，未部署时钟环境。
6. **启发式基线选取范围窄**：仅与简单固定预算和简单启发式（贪心、中峰）比较，未与更先进的元计算分配方法（如 Metacontrol, Thinker, Dynamic Thinker）或在线学习分配策略对比。
7. **缺乏对随机性/连续时间的完整处理**：原文承认实际部署中 MCTS 运行时间有轻微随机性，属于连续时间 SMDP，但训练中按确定性离散处理，虽实验显示迁移良好，但理论上可能有差距。
