## 摘要
代理强化学习可能引发工具滥用，即模型过度使用外部工具，即使对于可通过内部推理解决的查询也是如此。现有方法通过统一的工具使用惩罚或硬限制来缓解此问题，这虽然降低了工具使用频率，但也可能抑制有用的工具辅助探索。我们提出EAPO（高效代理策略优化框架），该框架学习选择性的工具使用。EAPO在每次滚动组中引入无工具轨迹，应用难度感知的奖励塑造来主要对简单查询惩罚冗余的工具调用，并使用置信度感知的令牌重新加权来改进策略学习。在九个数学和知识密集型推理基准测试中，EAPO在Qwen2.5-3B、Qwen2.5-7B和Llama3.1-8B上一致地提高了准确率-效率权衡。与GRPO相比，EAPO将平均性能分别提高了10.45%、7.27%和9.69%，同时将平均工具调用次数分别减少了18.33%、18.33%和24.59%。这些结果表明，代理可以学会何时不使用工具，而不会影响工具集成的推理。

## Abstract
Agentic reinforcement learning can induce tool abuse, where models overuse external tools even for queries solvable by internal reasoning. Existing approaches mitigate this issue with uniform tool-use penalties or hard limits, which reduce tool frequency but may also suppress useful tool-assisted exploration. We propose EAPO, an Efficient Agentic Policy Optimization framework that learns selective tool use. EAPO introduces tool-free trajectories into each rollout group, applies difficulty-aware reward shaping to penalize redundant tool calls mainly on easier queries, and uses confidence-aware token reweighting to improve policy learning. Across nine mathematical and knowledge-intensive reasoning benchmarks, EAPO consistently improves the accuracy efficiency trade-off on Qwen2.5-3B, Qwen2.5-7B, and Llama3.1-8B. Compared with GRPO, EAPO improves average performance by 10.45%, 7.27%, and 9.69%, while reducing average tool calls by 18.33%, 18.33%, and 24.59%, respectively. These results show that agents can learn when not to use tools without compromising tool-integrated reasoning.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：在大语言模型（LLM）的智能体强化学习（Agentic RL）训练中，模型容易产生**工具滥用（tool abuse）**——即使对于仅需内部推理即可回答的简单问题（如“1+1=？”），模型也会频繁调用外部工具（如Python解释器、搜索引擎），导致不必要的计算、延迟和部署成本。
- **现有方法的不足**：现有方法（如OTC、R1-Searcher++）采用统一工具惩罚或硬限制（如固定调用次数上限），虽然能降低工具调用频率，但可能**抑制对真正困难问题的有用探索**，从而损害工具辅助推理的核心价值。
- **研究目标**：不是让模型减少工具调用，而是**学会选择性使用工具**（只在必要时调用），实现精度与效率的更好权衡。

## 2. 方法论：核心思想、关键技术细节、公式与算法流程（文字说明）

**核心思想**：通过引入**无工具轨迹**提供“能否不用工具”的信号，在此基础上设计**难度感知奖励**和**置信度感知权重**，使模型能够在简单问题上主动避免工具，在困难问题上保留工具探索能力。

### 关键技术细节

#### (1) 效率感知 rollout（Efficiency-Aware Rollout）
- 在每个 rollout 组（共 M 条轨迹）中，保证至少 **n 条轨迹不使用任何工具**（通过修改提示词、禁用工具执行实现）。
- 定义效率系数：
  ```
  ε(T_i) = exp(-β * (c_i - c_min))
  ```
  - c_i：第 i 条轨迹的工具调用次数
  - c_min：正确轨迹中的最小工具调用次数（若无正确轨迹则设为0）
  - β：超参数，控制惩罚强度
  - 对于无工具轨迹，设 c_i = c_min

#### (2) 难度感知奖励塑造（Difficulty-Aware Reward Shaping）
- 定义问题难度 d(q)：
  ```
  d(q) = 1 - (1/M) * Σ I(T_i)
  ```
  I(T_i)=1 表示轨迹正确，否则为0。难度值越大说明问题越难。
- 调整后的奖励：
  ```
  R̃_i = [d(q) + (1 - d(q)) * ε(T_i)] * R_i
  ```
  - 对于简单问题（d(q) 小），效率系数 ε 的影响更大，从而惩罚过度使用工具。
  - 对于困难问题（d(q) 大），接近原始奖励，保留探索空间。
- 基础奖励 R_i 采用层次化奖励：格式正确时使用准确率（数学任务为0/1，QA任务可为F1或Recall），格式错误时设为 -1。

#### (3) 置信度感知优势重加权（Confidence-Aware Advantage Reweighting）
- 在每个 turn（工具调用的一轮）内，对参与策略学习的令牌计算置信度（log概率）。
- 设置重加权比例 ρ（例如0.2）：
  - 若轨迹正确：对**低置信**（低于 ρ 分位数）令牌赋予权重 W_pos（上提）
  - 若轨迹错误：对**高置信**（高于 (1-ρ) 分位数）令牌赋予权重 W_neg（下压）
  - 其他令牌权重为1。
- 重加权后的优势：A_{i,u,t} = ω_{i,u,t} * A_i（A_i 为轨迹级优势）。
- 目标函数：最大化期望重加权优势，使用策略梯度优化。

### 算法流程（文字描述）
1. 采样小批量查询。
2. 对每个查询，生成 n 条无工具轨迹 + M-n 条有工具轨迹。
3. 计算每条轨迹的正确性、工具调用次数、基础奖励。
4. 计算问题难度 d(q) 和最小工具调用 c_min。
5. 计算效率系数并调整奖励。
6. 从调整后的奖励计算轨迹优势。
7. 对每个参与学习的令牌，根据置信度计算权重 ω。
8. 使用重加权优势计算策略梯度，更新策略参数，并加入KL正则化。

## 6. 主要结论与发现

- **EAPO在精度-效率权衡上显著优于基线**：与GRPO相比，平均性能提升7-10%，工具调用减少18-25%（Qwen3B/7B和Llama8B）。

- **难度感知机制有效**：在训练早期（准确率低时），β影响较小，避免过早限制探索；后期准确率高时，惩罚有效抑制冗余调用。
