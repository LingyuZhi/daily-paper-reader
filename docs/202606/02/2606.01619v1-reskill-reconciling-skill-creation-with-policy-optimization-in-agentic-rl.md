## 摘要
智能体强化学习使得大规模语言模型代理能够从环境奖励中持续改进，但由此产生的策略并不会系统地积累可跨任务泛化的可复用策略。模块化技能能够提供这种可复用策略，然而现有的技能增强型强化学习方法将技能创建与策略优化分离，存在采用与不断演化的策略相冲突的技能的风险。受Anthropic的Skill Creator启发，我们提出了ReSkill，一种将技能演化与策略学习相协调的强化学习循环内技能创建框架。ReSkill利用GRPO的分组结构，仅需少量额外开销即可自然嵌入三种机制：（1）基于断言的技能创建器，从过去经验中诊断失败，并提出有条件的、基于触发的技能修订；（2）组内轨迹采样，实现对技能版本的可控比较，捕获最能支持策略持续学习的版本；（3）自适应折扣的汤普森采样，在策略演化过程中平衡技能版本选择中的探索与利用。在多个领域，ReSkill持续优于现有的基于记忆和技能的强化学习方法，在未见任务上提升最为显著。对技能生命周期的分析表明，随着策略的改进，技能被自动创建、测试、完善和修剪，展现了协调的技能-策略共同演化。

## Abstract
Agentic reinforcement learning (RL) enables LLM agents to improve continuously from environment rewards, yet the resulting policies do not systematically accumulate reusable strategies that generalize across tasks. Modular skills can provide such reusable strategies, yet existing skill-augmented RL methods decouple skill creation from policy optimization, risking adopting skills that conflict with the evolving policy. Inspired by Anthropic's Skill Creator, we introduce ReSkill, an RL-in-the-loop skill creation framework that reconciles skill evolution with policy learning. ReSkill exploits the group-wise structure of GRPO to naturally embed three mechanisms with only marginal additional overhead: (1) an assertion-driven skill creator that diagnoses failures from past experience and proposes conditional, trigger-based skill revisions; (2) within-group rollout sampling that enables controlled comparison of skill versions, capturing which version best supports the policy's ongoing learning; and (3) Thompson Sampling with adaptive discounting to balance exploration and exploitation in skill version selection as the policy evolves. Across several domains, ReSkill consistently outperforms existing memory and skill-based RL methods, with the largest gains on unseen tasks. Analysis of the skill lifecycle shows skills being automatically created, tested, refined, and pruned as the policy improves, demonstrating reconciled skill-policy co-evolution.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：现有的大语言模型（LLM）智能体在强化学习（RL）训练中虽然能从奖励信号中不断改进，但学到的策略无法系统地积累可跨任务泛化的可复用策略。模块化技能（modular skills）可以封装可复用的任务策略，但现有技能增强型 RL 方法（如 SkillRL、EvolveR、INSPO 等）将技能创建与策略优化脱钩，外部生成的技能可能与演化中的策略冲突，导致性能下降。受 Anthropic 的 Skill Creator（人工参与、静态策略下测试技能）启发，作者希望将技能创建过程嵌入到 RL 训练循环中，实现技能与策略的协同进化（co-evolution）。
- **整体含义**：ReSkill 通过在设计上使每个 rollout 同时服务于策略优化、失败诊断和技能版本评估（三重职责），在不增加额外 rollout 预算的前提下，让技能随着策略学习而自动创建、测试、精炼和修剪，从而提升智能体的泛化能力和学习效率。

## 2. 方法论

### 核心思想
- 利用 GRPO（Group Relative Policy Optimization）的分组结构：每个训练步骤采样的 G 个 rollout 共享同一任务和同一策略，唯一变量是技能版本（新 vs. 旧），从而提供天然的可控对比环境。
- 嵌入三种机制：
  - **断言驱动的技能创建器（Assertion-driven skill creator）**：从训练轨迹中诊断失败模式，提出有条件的、基于触发的技能修订。
  - **组内 rollout 采样（Within-group rollout sampling）**：每个 rollout 分配不同的技能版本，在 GRPO 分组内进行受控比较。
  - **自适应折扣的汤普森采样（Thompson Sampling with adaptive discounting）**：平衡技能版本选择的探索与利用，并自动折扣随着策略演化而过时的过往证据。

### 关键技术细节
1. **RL 训练与组内技能测试**：
   - 每个 rollout 的版本`v_i`从汤普森采样后验中抽取，轨迹在对应技能库`S_{v_i}`下生成。
   - 扩展的 GRPO 目标函数（式 2）：`J_{RESKILL}(θ) = E_{q∼D, v_i∼TS, τ_i∼π_{θ_old}} [ L_{GRPO}(θ; {τ_i}) ]`，其中绿色部分表示 ReSkill 引入的组件。
   - 每个 rollout 三重职责：① 提供 GRPO 梯度用于策略优化；② 补全轨迹存入经验水库供技能创建；③ 更新汤普森采样后验用于接受/拒绝决策。

2. **循环内技能创建（RL-in-the-loop skill creation）**：
   - **经验水库**：持续收集成功-失败轨迹对，始终反映当前行为。
   - **条件技能加载**：每个技能包含内容（何时使用、动作指导、示例）和触发条件（general/beginning/action-pattern），只有触发条件满足的技能才被加载到上下文中。
   - **基于断言的失败分级**：维护动态断言集`A`，对所有轨迹计算每断言通过率，LLM 分析器生成排序的失败诊断并更新断言集。
   - **历史知识驱动的技能修订**：给定当前技能库、失败诊断和过往提议历史，技能创建器提出一组 ADD/MODIFY/DELETE 操作形成候选新版本`S_{new}`，并在测试前验证触发覆盖率。

3. **RL 引导的技能演化（Thompson Sampling）**：
   - **版本分配**：两个 arm（旧/新）成功率服从 Beta 分布，通过蒙特卡洛估计`Pr(p_new > p_old)`，设`π_t(new) = Clip(概率, ε_ts, 1-ε_ts)`，然后独立为每个 rollout 抽样版本（式 3）。
   - **策略感知的后验更新**：考虑策略演化，引入自适应折扣权重`w_t = 1/(1 + n_t/M)`，在每步更新后验时缩水旧证据（式 4）。`M`为记忆参数，通过已完成的评估周期上最大化序列预测似然来自动估计。
   - **接受/拒绝决策**：周期结束时，若期望`E[p_new] > E[p_old]`则接受新版本，否则回退到旧版本，立即开始下一进化周期。

### 算法流程（文字描述）
- 初始化策略、技能库、经验水库和后验。
- 对每个训练步 t：
  - 采样任务 q，根据汤普森采样计算版本分配概率，为每个 rollout 分配版本。
  - 加载对应技能库，生成 G 个 rollout，计算奖励，更新策略（J_RESKILL）。
  - 将轨迹加入经验水库。
  - 根据当前步的观测更新后验（折扣后再加新计数）。
  - 若 t 是进化频率的整数倍：执行接受/拒绝决策（更新技能库），然后运行技能创建（从水库诊断失败 → 提出新候选版本），重置后验开始新测试周期。

- ReSkill 在所有基准上一致优于所有对比方法，在 ALFWorld 和 Search 上分别比最强技能基线（SkillRL）高出 5.9%/6.9%（4B）和 6.9%/2.3%（8B）。

## 7. 优点

## 8. 不足与局限

- **进化频率需调参**：ALFWorld 上性能对 Ke 敏感（Ke=20 时下降约 5%），虽在 Search 上稳定，但最优值可能因训练步数而异。作者指出自适应进化调度是未来方向。
- **技能调用方式限制**：当前触发设计将技能加载到上下文中，由模型自主决定是否应用，可能引起“元错误”（如将技能名当作环境命令）。工具调用形式可提供更精细控制但会增加推理开销。
- **技能容量负担**：小型模型有时会混淆技能指引与环境动作，或过度应用不适用的技能，表明需要训练模型将技能视为推理时指导而非动作时指令。
- **实验覆盖不够完整**：
  - 未提供与 O1/GPT-4o 等更强闭源模型的直接对比（仅对比了 Claude 4.5 Sonnet 作为参考）。
  - 仅在 ALFWorld 和 Search 上进行了 4B/8B 双模型尺度验证，其他三个扩展域仅用 4B 单一模型。
  - 未报告训练时间的绝对数值（小时），只提供了相对比值。
  - 未进行更细粒度的鲁棒性分析（如不同随机种子间方差讨论较多但未提供完整分布图）。
- **适用性局限**：方法依赖于 GRPO 的组结构，若使用其他 RL 算法（如 PPO），则组内比较机制需要重新设计。
- **创建器依赖 LLM**：技能创建依赖外部 LLM（默认 Claude 4.5），虽然消融表明即使使用基础模型仍优于所有基线，但跨不同创建器能力的鲁棒性尚待进一步验证。
