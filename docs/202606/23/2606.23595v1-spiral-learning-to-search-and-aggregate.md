## 摘要
在测试时，通过扩展推理计算量并使用不同的原语——轨迹内的顺序推理、独立采样的并行轨迹以及将多个推理轨迹聚合为最终响应，语言模型的推理能力可以得到显著提升。然而，在后训练阶段，语言模型仅针对单一轨迹内的顺序推理进行了优化。我们提出了顺序-并行-聚合强化学习（SPIRAL），该框架使语言模型能够将三种原语作为统一推理计算管线的一部分进行训练。具体来说，语言模型首先并行采样一组独立轨迹，每个轨迹通过顺序思维链推理生成，然后基于这些轨迹生成最终的聚合轨迹；所有组件根据最终聚合响应的奖励进行端到端优化。为了训练这一系统，SPIRAL使用集合强化学习来教模型生成一组对聚合器集体有用的轨迹，并使用标准强化学习来教模型将集合聚合为改进的最终响应。我们在推理任务上的实验表明，SPIRAL能有效随推理计算量扩展，在三种计算原语都进行扩展时，其扩展效率比GRPO高出多达11倍，性能高出15%。

## Abstract
Language model reasoning can be substantially improved at test time via scaffolds that scale inference compute across different primitives -- sequential reasoning within a trace, independently sampled parallel traces, and aggregation of multiple reasoning traces into a final response. During post-training, however, language models are optimized only for sequential reasoning within a single trace. We introduce Sequential-Parallel-Aggregative Reinforcement Learning (SPIRAL), a framework in which a language model is trained to use all three primitives, as part of a unified inference compute pipeline. Concretely, the language model first samples a set of independent traces in parallel, each produced through sequential chain-of-thought reasoning, and then generates a final aggregation trace conditioned on those traces; all components are optimized end-to-end against the reward of the final aggregated response. To train this system, SPIRAL uses set reinforcement learning to teach models to produce a set of traces that are collectively useful for an aggregator and standard reinforcement learning to teach models to aggregate the set into improved final responses. Our experiments on reasoning tasks show that SPIRAL effectively scales with inference compute, outperforming GRPO by up to 11$\times$ scaling efficiency and 15% higher performance when all three compute primitives are scaled.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **现有问题**：当前语言模型的后训练（post-training）仅针对单一轨迹内的顺序推理（sequential chain-of-thought）进行优化；而在测试时，研究者常通过并行采样多条轨迹和聚合多条轨迹来提升推理性能。这种训练与测试的失配导致模型无法有效协调三种基本推理原语（顺序、并行、聚合），常需手工设计复杂的“脚手架”（scaffolds）来组织这些计算。
- **研究动机**：消除这一失配，让模型在训练阶段就学会如何最优地使用所有三种原语（顺序推理、并行探索、聚合合成），从而在测试时更有效地利用推理计算资源。
- **整体含义**：SPIRAL 提出一个统一的强化学习框架，端到端优化模型的搜索（search）和聚合（aggregation）能力，使模型能够自主发现更优的探索-利用策略，超越手工设计的启发式方法。

### 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

#### 核心思想
- 将推理过程建模为两级生成：
    1. **搜索轨迹（search traces）**：给定问题 \(x\)，模型 \(\pi_\theta\) 独立并行采样 \(n\) 条顺序推理轨迹 \(y_1,\dots,y_n\)。
    2. **聚合轨迹（aggregation trace）**：模型基于问题 \(x\) 和所有搜索轨迹 \(y_1,\dots,y_n\)，生成最终的聚合解 \(y^*\)。
- 整个管线的优化目标为：\(\max_{\theta,\phi} \mathbb{E}_{y_{1:n}\sim\pi_\theta(\cdot|x)} \mathbb{E}_{y^*\sim\pi_\phi(\cdot|x,y_{1:n})}[r(x,y^*)]\)，其中 \(\theta=\phi\)（同一模型）或可分离。
- **关键创新**：优化该目标得到两个梯度项：
    - **Term 1（集合RL梯度）**：更新搜索轨迹，使生成的一组轨迹对聚合器整体有用；
    - **Term 2（标准RL梯度）**：更新聚合轨迹，使其能有效综合给定集合得到正确回答。

#### 关键技术细节
- **集合强化学习（Set RL）**：用于优化搜索轨迹。对每个问题采样 \(N_1\) 条搜索轨迹，从中无放回均匀抽取 \(K\) 个大小为 \(n\) 的子集，对每个子集采样 \(N_2\) 条聚合轨迹，计算子集平均奖励作为集合级目标 \(\hat{f}_{\text{spiral}}(x,\mathcal{G})\)。定义集合基线 \(\hat{f}_{\text{spiral}}(x)\) 和集合优势 \(A^\sharp(x,\mathcal{G}; f_{\text{spiral}})\)，再通过边际集合优势 \(A^\sharp_{\text{marg}}(x,y_i)\) 为每个搜索轨迹分配信用。
- **标准RL**：用于优化聚合轨迹，每个集合内部使用中心化优势 \(A(x,y^{\mathcal{G}_i}_j) = r(x,y^{\mathcal{G}_i}_j) - \frac{1}{N_2}\sum_k r(x,y^{\mathcal{G}_i}_k)\)。
- 最终梯度为两者之和，通过 REINFORCE 类算法（文中使用 Tinker 实现）更新模型参数。

#### 算法流程简述
1. 对每个问题 \(x\)，采样 \(N_1\) 条搜索轨迹；
2. 随机构造 \(K\) 个大小为 \(n\) 的子集（无放回）；
3. 对每个子集采样 \(N_2\) 条聚合轨迹并计算奖励；
4. 计算各集合的平均奖励、集合基线、集合优势；
5. 计算每个搜索轨迹的边际集合优势；
6. 计算集合RL梯度（搜索轨迹）和标准RL梯度（聚合轨迹），合并更新模型。

- **基模型**：Qwen3-4b-Instruct-2507。
- **对比方法**：

### 7. 优点

- **统一训练框架**：首次将顺序、并行、聚合三种推理原语在训练阶段端到端优化，弥合训练-测试鸿沟。
- **创新的信用分配**：利用集合RL为搜索轨迹提供共享学习信号，使模型学会生成对聚合有用的“集体”轨迹，而非仅追求单迹正确。
- **低方差基线**：聚合轨迹使用集合内中心化优势，降低了方差，稳定训练。
- **可扩展性**：虽然训练时固定集合大小，测试时可灵活扩展并行和聚合计算（如递归自聚合），模型表现出良好的泛化能力。
- **无需手工设计**：模型自动习得搜索、验证、精炼行为，减少人工脚手架依赖。
