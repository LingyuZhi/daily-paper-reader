## 摘要
智能体强化学习可能导致工具滥用，即模型过度使用外部工具，即使对于可以通过内部推理解决的查询也是如此。现有方法通过统一的工具使用惩罚或硬限制来缓解这一问题，这虽然降低了工具使用频率，但也可能抑制有用的工具辅助探索。我们提出了EAPO，一个高效的智能体策略优化框架，用于学习选择性工具使用。EAPO在每个回合组中引入无工具轨迹，应用难度感知的奖励塑造来惩罚主要在简单查询上的冗余工具调用，并使用置信度感知的令牌重加权来改进策略学习。在九个数学和知识密集型推理基准测试中，EAPO一致地改善了Qwen2.5-3B、Qwen2.5-7B和Llama3.1-8B上的准确率-效率权衡。与GRPO相比，EAPO平均性能分别提升了10.45%、7.27%和9.69%，同时平均工具调用次数分别减少了18.33%、18.33%和24.59%。这些结果表明，智能体可以学会何时不使用工具，而不会损害工具集成的推理能力。

## Abstract
Agentic reinforcement learning can induce tool abuse, where models overuse external tools even for queries solvable by internal reasoning. Existing approaches mitigate this issue with uniform tool-use penalties or hard limits, which reduce tool frequency but may also suppress useful tool-assisted exploration. We propose EAPO, an Efficient Agentic Policy Optimization framework that learns selective tool use. EAPO introduces tool-free trajectories into each rollout group, applies difficulty-aware reward shaping to penalize redundant tool calls mainly on easier queries, and uses confidence-aware token reweighting to improve policy learning. Across nine mathematical and knowledge-intensive reasoning benchmarks, EAPO consistently improves the accuracy efficiency trade-off on Qwen2.5-3B, Qwen2.5-7B, and Llama3.1-8B. Compared with GRPO, EAPO improves average performance by 10.45%, 7.27%, and 9.69%, while reducing average tool calls by 18.33%, 18.33%, and 24.59%, respectively. These results show that agents can learn when not to use tools without compromising tool-integrated reasoning.

- **现有方法不足**：当前缓解策略（如统一惩罚工具使用或设定硬调用次数限制）虽然能减少工具调用次数，但可能抑制在真正困难问题上的有用工具探索，损害性能。
- **论文目标**：提出一种让模型**学会选择性使用工具**的方法，即不仅能使用工具，还要知道**何时不需要**，从而实现准确率与效率的更好权衡。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 核心思想
提出 **EAPO（Efficient Agentic Policy Optimization）** 框架，由三个模块组成：
1. **效率感知的Rollout**：在每组采样轨迹中确保至少 \(n\) 条轨迹**不使用任何工具**，提供模型能否靠内部推理解决查询的直接对比信号。
2. **难度感知的奖励塑形**：根据查询的难度动态调整奖励，对简单查询上的冗余工具调用进行惩罚，但对困难查询保留工具探索空间。
3. **置信度感知的Token重加权**：根据模型生成令牌时的置信度调整学习权重，在正确轨迹中强调低置信令牌，在错误轨迹中抑制高置信令牌。

### 2.2 关键技术细节与公式

#### (1) 效率系数（Efficiency Coefficient）
对于轨迹 \(T_i\)，设工具调用次数为 \(c_i\)，正确轨迹中最小调用次数为 \(c_{\text{min}}\)（若无正确轨迹则设为0），效率系数为：
\[
\varepsilon(T_i) = \exp(-\beta(c_i - c_{\text{min}}))
\]
若轨迹无工具调用，则 \(c_i = c_{\text{min}}\)。\(\beta\) 控制惩罚强度。

#### (2) 难度定义
查询 \(q\) 的难度：
\[
d(q) = 1 - \frac{1}{M} \sum_{i=1}^{M} \mathbb{I}(T_i)
\]
其中 \(\mathbb{I}(T_i)=1\) 当轨迹正确，\(M\) 为组内轨迹总数。

#### (3) 调整后的奖励
\[
\tilde{R}_i = \left[d(q) + (1-d(q))\varepsilon(T_i)\right] R_i
\]
\(R_i\) 是原始奖励（正确且格式良好时取准确率，否则为 -1）。当查询难度低（\(d(q)\) 小）时，效率系数主导惩罚；当难度高时，效率系数贡献小，保护有用工具使用。

#### (4) 置信度感知的Token重加权
对轨迹中每个参与策略学习的令牌 \(s_{i,u,t}\)，计算置信度 \(C_{i,u,t} = \log p_t(y_t)\)。按以下规则分配权重 \(\omega_{i,u,t}\)：
- 若轨迹正确且令牌置信度低于 \(\rho\) 分位数：\(\omega = W_{\text{pos}}\)
- 若轨迹错误且令牌置信度高于 \(1-\rho\) 分位数：\(\omega = W_{\text{neg}}\)
- 其他情况：\(\omega = 1\)

最终重加权优势 \(A_{i,u,t} = \omega_{i,u,t} A_i\)，用于策略梯度更新。

### 2.3 算法流程（文字描述）
1. 每步迭代：采样 \(M\) 条轨迹，其中至少 \(n\) 条无工具（禁用工具执行），其余 \(M-n\) 条允许工具使用。
2. 计算每条轨迹的正确性、工具调用次数、原始奖励。
3. 计算查询难度 \(d(q)\)，最小工具调用 \(c_{\text{min}}\)，效率系数 \(\varepsilon(T_i)\)，调整奖励 \(\tilde{R}_i\)。
4. 从调整后奖励计算组内优势 \(A_i\)。
5. 对每个策略学习用令牌，根据置信度计算权重 \(\omega\)，得到重加权优势。
6. 使用带KL正则化的重加权策略梯度更新策略。
