## 摘要
LLM智能体运行在两种不同的模式下：可进行强化学习（RL）的开源权重智能体，以及其行为必须在测试时纯粹通过控制的黑盒智能体。尽管黑盒智能体通常由最先进的专有LLM支持，但仅通过API访问排除了参数级优化，使得大多数RL方法无法适用。为解决这一限制，我们转向利用RL与贝叶斯推理之间已知的等价性。我们提出主体蒙特卡洛（Agentic Monte Carlo, AMC），直接从黑盒智能体的最优策略中采样，而非通过RL训练。最优策略是轨迹上的后验分布，其先验定义为固定的黑盒LLM智能体。我们采用序贯蒙特卡洛方法，通过学习一个价值函数来引导智能体，同时保持底层黑盒模型不变，从而从该后验中采样。我们在AgentGym基准测试的三个不同环境中验证了AMC，结果表明，随着我们方法测试时计算量的增加，AMC相较于提示基线有显著改进，甚至优于群体相对策略优化（GRPO）。AMC证明了在原则上对黑盒LLM智能体执行类似RL优化的可行性。代码已在 https://github.com/layer6ai-labs/Agentic-Monte-Carlo 上开源。

## Abstract
LLM agents operate in two distinct regimes: open-weight agents amenable to reinforcement learning (RL) and black-box agents whose behaviour must be controlled purely at test time. Although black-box agents are often backed by state-of-the-art proprietary LLMs, API-only access precludes parameter-level optimization, rendering most RL methods inapplicable. To address this limitation, we turn to a known equivalence between RL and Bayesian inference. We propose Agentic Monte Carlo (AMC) to directly sample from the optimal policy of a black-box agent rather than training it through RL. The optimal policy is a posterior over trajectories whose prior we define as the fixed black-box LLM agent. We employ Sequential Monte Carlo to sample from this posterior by learning a value function to steer the agent while leaving the underlying black-box model unchanged. We validate AMC on three diverse environments from the AgentGym benchmark, demonstrating significant improvements over prompting baselines and even outperforming Group Relative Policy Optimization (GRPO) as we scale the test-time compute of our method. AMC demonstrates the feasibility of performing principled RL-style optimization of black-box LLM agents. Code is available at https://github.com/layer6ai-labs/Agentic-Monte-Carlo

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：当前最先进的 LLM 智能体（如 GPT-5、Gemini 3、Claude 4.6）大多仅以黑盒 API 形式提供，无法进行参数级优化。传统的强化学习（RL）方法（如 PPO、GRPO）需要白盒访问模型权重计算策略梯度，因此对黑盒智能体完全不适用。研究者只能通过 Prompt Engineering 或微调开源替代模型来调整行为，无法真正进行 RL 风格的最优策略搜索。
- **整体含义**：论文旨在证明即使在纯黑盒 API 访问限制下，也能通过统计推断的方法模拟 RL 效果，实现对黑盒智能体行为的原则性优化。这为 GPU 受限或只能使用闭源模型的场景提供了新范式。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

- **核心思想**：利用 KL 正则化 RL 与贝叶斯推断之间的等价性，将寻找最优策略的问题转化为从贝叶斯后验分布中采样。该后验定义为：π*(s₀:T) ∝ π(s₀:T) exp(r(s₀:T)/β)，其中 π 是固定的黑盒 LLM 先验策略，r 是累积奖励，β 是正则化系数。这样，无需修改模型参数，只需从该后验中采样即可得到近似最优策略。
- **关键技术细节**：
  - **序贯蒙特卡洛（SMC）方法**：采用序贯重要性重采样（SIR）算法，并行运行 N 条轨迹，每个时间步根据重要性权重进行重采样，剪枝低质量轨迹、繁殖高质量轨迹，使轨迹集逐渐逼近后验分布。
  - **重要性权重递归分解**：通过引入软价值函数 V(s_t) = log E[exp( Σ_{τ=t}^T r(s_τ)/β ) | s_t]，将权重更新简化为：w_t = w_{t-1} · exp( V(s_t) - V(s_{t-1}) + r(s_{t-1})/β )。
  - **学习价值函数**：由于真实 V(s_t) 未知，通过收集先验策略 π 的蒙特卡洛轨迹，使用一个轻量级 Transformer 模型 fθ 回归未来累计奖励（r(s_{t+1:T})），并设置 V_θ(s_t) = f_θ(s_t) + r(s_t)。训练目标为 MSE 损失，利用单一轨迹近似软最大值。
  - **AMC 整体流程**：Step 1: 从先验策略 π 采样 M 条轨迹，训练价值函数 V_θ；Step 2: 在推理阶段，使用 SIR 算法运行 N 条轨迹，在每个（选定的）重采样步根据 V_θ 计算权重并重采样；Step 3: 最终选择累积奖励最高的轨迹作为输出。
