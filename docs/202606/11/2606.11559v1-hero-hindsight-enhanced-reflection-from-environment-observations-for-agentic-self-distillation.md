## 摘要
强化学习通常通过轨迹的终端结果来提升多轮智能体的能力，这使得难以确定每一中间轮的信用分配。最近的在策略自蒸馏方法通过自教师将特权反馈转化为密集的标记级监督，提供了一种有前景的替代方案。我们的研究受到一个意外现象驱动：当朴素地将该范式扩展到多轮设置时，性能出现下降，我们将其归因于特权反馈（如成功轨迹或终端结果）与学生当前决策上下文之间缺乏对齐。我们提出了 HERO，一种 hindsight 增强的自蒸馏框架，它使用下一环境观察作为局部对齐的反馈。每次 rollout 后，HERO 反思已完成的交互，将每个观察转化为紧凑的回合级诊断，捕获关于原始动作的可操作反馈，例如其必要性、有效性或失败原因。在 TauBench 和 WebShop 上，HERO 相比仅基于环境反馈的自蒸馏和 GRPO，提高了任务成功率并减少了不必要的回合。在训练回合预算有限的情况下，HERO 尤其有效，因为此时成功的 rollout 很少，而 GRPO 提供的奖励对比信号较弱。

## Abstract
Reinforcement learning typically improves multi-turn agent capabilities through the terminal outcome of the trajectories, which makes it difficult to determine credit assignments for each intermediate turns. Recent on-policy self-distillation methods offer a promising alternative by converting privileged feedback into dense token-level supervision through a self-teacher. Our study is motivated by the unexpected performance degradation observed when naively extending this paradigm to multi-turn settings, which we attribute to a lack of alignment between privileged feedback, such as successful trajectories or terminal outcomes, and the student's current decision context. We introduce HERO, a hindsight-enhanced self-distillation framework that uses next environment observations as locally aligned feedback. After each rollout, HERO reflects on the completed interaction to convert each observation into a compact turn-level diagnosis, that captures actionable feedback about the original action such as its necessity, validity or failure cause. On TauBench and WebShop, HERO improves task success and reduces unnecessary turns over environment-feedback-only self-distillation and GRPO. It is especially effective under limited training turn budgets, where successful rollouts are rare and GRPO provides weak reward-contrast signals.

### 1. 核心问题与整体含义（研究动机和背景）

- **问题**：强化学习（RL）在多轮智能体任务中通常仅依赖轨迹的终端结果（如二元奖励）进行优化，这导致**中间步骤的信用分配困难**。例如，即使最终成功的轨迹也可能包含冗余或无效的工具调用，但标量奖励无法提供局部反馈。
- **现有局限**：近年来提出的**在策略自蒸馏**方法（如 SDPO、OPSD）在单轮任务（如竞赛编程）中表现优异，通过将特权反馈（如成功演示、环境观察）转化为密集的 token 级监督。然而，**朴素地将这些方法扩展到多轮设置会导致性能下降**，原因是“全轨迹演示教师”在早期回合中使用了学生不可见的未来动作，造成教师-学生上下文不匹配。
- **研究动机**：在多轮交互中，最自然、最及时的局部反馈信号是**每个动作之后立即返回的下一环境观察**。如何有效利用这一信号进行训练，成为本文的核心问题。

### 2. 方法论

- **核心思想**：提出 **HERO（Hindsight-Enhanced Reflection from Environment Observations）** 框架，通过 **后见反思** 将完整轨迹信息压缩为每个回合的紧凑诊断提示，然后以此为条件构建自教师，产生密集的 token 级监督，从而避免直接使用完整的 off-policy 成功轨迹。
- **关键技术细节**：
  1. **学生策略采样**：学生使用标准因果历史生成多轮轨迹 `τ`。
  2. **后见反射器**：rollout 结束后，反射器 R 查看完整轨迹和最终奖励，输出结构化回合级提示 `ht`，包括自然语言诊断 `dt` 和可选的校正动作 `ˆat`。若某回合无可反馈信号，则置 `ht = ∅`。
  3. **自教师构建**：对于每个回合 t，教师上下文为 `(Ht, ot, R(τ), ht)`，使用固定模板（图 3，包含原始任务、轨迹结果、环境反馈和反射诊断）提示。教师对学生原始动作的每个 token 计算分布 `pteat,k`，并与学生分布 `pstut,k` 计算 JSD 损失。
  4. **优化目标**：`LHERO(θ) = Eτ[ (1/|I(τ)|) Σt∈I(τ) ℓt ]`，其中 `ℓt` 为回合内所有 token 的 JSD 均值。仅对非空提示回合计算损失。
- **关键设计差异**：与“全演示教师”不同，HERO 不将完整未来轨迹暴露给教师，而是只提供压缩后的局部提示，从而避免上下文不匹配。并且即使全部轨迹失败，只要反射器检测到可纠正的局部错误，HERO 仍可提供非零监督信号（GRPO 在全部失败时会因组内奖励无差异而失去学习信号）。

### 3. 实验设计

- **数据集与场景**：
  - **TauBench-Retail**（训练 + 验证，500/115 个实例）：多轮客服任务，结构化 API 调用。
  - **WebShop**（训练 + 测试，6219/691 个实例）：基于 web 的多步购物，自然语言指令的产品选择。
  - **TauBench-Airline**（仅测试 OOD，50 个实例）：票务预订场景，用于评估跨领域泛化。
- **对比方法**：
  - **Base**：原始指令微调模型，无训练。
  - **Environment Feedback Only**：教师仅使用原始环境观察作为特权上下文。
  - **Full-Demo Privileged Teacher**：教师使用完整成功轨迹作为特权上下文（多轮 SDPO 朴素扩展）。
  - **GRPO**：基于组相对优势的强化学习方法。

### 6. 主要结论与发现

- **HERO 在所有设置下优于环境反馈自蒸馏和 GRPO**，在 Qwen3-4B 上 TauBench-Retail 成功率 34.7% vs GRPO 33.3%，WebShop 68.9% vs 65.2%；在 30B 模型上优势更明显。
- **HERO 显著降低不必要的回合数**，尤其在 TauBench 上，从 GRPO 的 13.4 回合降至 9.6 回合（Retail），效率提升约 28%。
- **全演示教师（SDPO 朴素扩展）性能低于基线甚至降级**，验证了教师-学生上下文不匹配的危害。
- **HERO 可从失败轨迹中学习**，在严格轮次预算下 GRPO 因奖励无差异而失效时，HERO 仍能提供有效监督。
- **单 rollout 学习可行**：HERO 在 G=1 时仍能取得接近 G=4 的性能，而 GRPO 在 G=1 时优势为 0。
- **泛化能力保持**：在 MMLU 等标准基准上无明显退化。

### 7. 优点

- **方法创新**：将后见反射压缩为回合级特权上下文，避免了全轨迹上下文不匹配，同时保留密集 token 级监督的优势。
- **对资源友好**：仅需单次 rollout 即可学习，训练速度比 GRPO 快约 2 倍，对实际高延迟环境（真实 API、模拟器）尤其重要。
- **鲁棒性**：在成功轨迹稀少、奖励信号稀疏的困难场景下仍能有效提升性能，适用于实际中大量失败场景的 agent 训练。
- **实验设计严谨**：覆盖不同模型规模、不同领域（包含 OOD）、多种消融和人工评估，结论可信度高。
- **实用价值**：通过减少不必要的工具调用提升了智能体的执行效率，且通用能力不受损。

### 8. 不足与局限

- **依赖模型自身反思能力**：HERO 的有效性依赖于模型能否在 hindsight 正确诊断错误。对于复杂数学推理或深度多步推理任务，模型可能无法识别根本原因，导致反射质量不高。
- **依赖指令遵循与上下文学习能力**：自教师必须正确理解反射提示并转化为 token 级偏好变化。因此更适合指令微调后的模型，对弱指令遵循模型可能效果不佳。
- **反射质量仍有改进空间**：自反射器的正确率约 58.9%（部分正确 9.4%），仍有 17.9% 的错误率和 13.8% 的空洞提示。更强的外部反射器（GPT-4o）可进一步提升性能，但增加了推理成本。
- **结论的泛化性**：实验仅覆盖 TauBench 和 WebShop 两类环境（API 调用和网页导航），未在数学推理、代码生成、对话等更广泛场景验证。此外，反射格式和教师模板需要针对不同环境定制。
