## 摘要
当前的具身机器人系统能够编写可执行的代码即策略程序、观察反馈并多次尝试修正行为，但它们仍然主要由任务驱动：只有在明确指令后才能获取可复用技能。我们研究趣味具身机器人学习，其中具身编码智能体在下游任务到来前，将自主导向的趣味互动作为持续技能学习阶段。我们引入了RATs，即为趣味时间技能获取而设计的机器人智能体团队。在趣味互动期间，RATs会提出新颖且可学习的探索性任务，规划并执行机器人代码策略，验证中间进展，诊断失败，通过密集的步骤级反馈进行重试，并将成功执行提炼为持久化的代码技能库。在测试时，智能体从该冻结技能库中复用相关技能以辅助解决新任务。在LIBERO-PRO和MolmoSpaces上的实验表明，趣味学习技能相比无趣味和随机趣味基线，在保留的下游任务上分别取得20.6和17.0个百分点的提升（相对于CaP-Agent0）。此外，通过简单地将所学技能检索至上下文，即可将其插入其他推理时的代码即策略智能体，在无需微调底层模型的情况下，分别使RoboSuite和真实世界迁移任务提升8.9和8.8个百分点。

## Abstract
Current agentic robot systems can write executable Code-as-Policy programs, observe feedback, and revise behavior across multiple attempts, but they remain largely task-driven: reusable skills are acquired only after explicit instructions. We study Playful Agentic Robot Learning, where an embodied coding agent uses self-directed play as a continual skill-learning stage before downstream tasks arrive. We introduce RATs, Robotics Agent Teams designed for play-time skill acquisition. During play, RATs proposes novel yet learnable exploratory tasks, plans and executes robot-code policies, verifies intermediate progress, diagnoses failures, retries with dense, step-level feedback, and distills successful executions into a persistent code skill library. At test time, the agent reuses relevant skills from this frozen library to help solve new tasks. Experiments in LIBERO-PRO and MolmoSpaces show that play-learned skills improve held-out downstream tasks over no-play and random-play baselines, with 20.6 and 17.0 percentage-point gains over CaP-Agent0 on LIBERO-PRO and MolmoSpaces, respectively. Moreover, the learned skills can be plugged into other inference-time Code-as-Policy agents by simply retrieving them into the context, improving RoboSuite and real-world transfer by 8.9 and 8.8 points, respectively, without finetuning the underlying model.

### 2. 方法论：核心思想、关键技术细节与算法流程

-   **核心思想**：通过一个名为RATs（Robotics Agent Teams）的多智能体系统，将“玩耍”转化为一个结构化的技能获取过程。系统在玩耍阶段没有外部奖励，而是由内在好奇心驱动，自主提议并实践“新颖且可学”的探索任务，从成功和失败的尝试中提取可复用的代码技能，形成一个持久化的智能体技能库。
-   **关键技术细节**：
    -   **多智能体团队架构**：RATs由三个协同工作的团队组成：
        1.  **任务提议者团队**：负责生成“玩耍”任务。它遵循**“金发姑娘原则”**，选择既不太简单也不太困难的任务，以最大化学习收益。任务选择公式为：`τt = arg max [N(τ) · F(τ)]`，其中`N(τ)`是对象-技能组合的新颖度（鼓励探索），`F(τ)`是能力边界（鼓励处于半熟悉状态，即成功率约0.5的任务）。
        2.  **执行团队**：负责完成任务。它由一个“编写-执行-验证-诊断”闭环构成，包含规划者、策略编写者、验证者（多级：步骤级、目标级）和失败诊断者。当诊断出某个子动作是瓶颈时，可以生成一个子智能体专项练习该子动作。
        3.  **记忆管理团队**：负责维护两个持久化存储：**技能库**（存储从成功执行中提炼出的可复用代码函数）和**失败记忆**（存储从失败中提炼出的简洁教训）。技能库中的技能有“实验性”、“已验证”和“弃用”三个可靠性等级，会基于后续表现进行动态调整。系统还会定期进行记忆整理，合并重复项，并主动提议新的辅助技能。
-   **算法流程**：
    1.  **玩耍阶段**：系统在与环境交互的N次迭代中，循环执行以下三步：
        -   **任务提议**：任务提议者根据当前场景、技能库和失败记忆，生成并选择一项“金发姑娘”任务。
        -   **任务执行**：执行团队通过多智能体协作，编写、执行、验证和诊断策略代码，尝试完成任务。如果失败，则根据诊断结果进行重试。
        -   **记忆更新**：记忆管理团队将成功执行提炼为技能存入技能库，将失败经历转化为教训存入失败记忆。
    2.  **测试阶段**：冻结技能库。面对新任务时，RATs从技能库中检索并调用相关技能来辅助完成任务。

### 8. 不足与局限

-   **实验覆盖偏差**：评估主要基于仿真环境，虽包含了一些真实世界实验，但任务较为简单（拾取和放置），且规模有限。结论在更复杂、更真实物理条件下的鲁棒性和泛化性有待验证。
-   **技能多样性受限**：玩耍阶段所能习得的技能受限于仿真环境所能提供的对象、动态和物理特性。在简单环境中玩耍可能无法习得解决更复杂现实世界问题所需的所有技能。
-   **技能复用风险**：当检索到的技能不完全适合当前任务时，可能会导致性能下降。论文也承认这一点，表明需要更好的检索和上下文感知选择机制。
-   **计算成本高昂**：RATs严重依赖大型语言模型（尤其是故障诊断和重试环节），计算开销和推理成本显著高于单次执行的CaP-Agent0。这部分在附录中已明确量化。
-   **依赖模型质量**：系统高度依赖视觉语言模型（VLM）的验证和诊断能力，模型的幻觉或判断失误可能会引入错误。
-   **低级控制API受限**：方法受限于原始的低级控制API（如点到点移动、夹爪开合），缺乏对高级、灵巧操纵技能的原生支持。这限制了它在需要精细处理任务上的应用。
