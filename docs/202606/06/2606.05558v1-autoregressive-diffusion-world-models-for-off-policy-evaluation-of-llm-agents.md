## 摘要
在多轮交互环境中评估大语言模型（LLM）智能体既昂贵又充满风险，因为需要在线环境交互。我们提出ADWM（自回归扩散世界模型），一种仅从预收集轨迹中评估新LLM智能体策略性能的框架。核心思想是学习一个潜在扩散世界模型，模拟环境对评估策略的响应，而无需在真实环境中执行该策略。现有的基于扩散的OPE方法通过联合扩散状态和动作的方式一次性指导完整轨迹，但这一假设在LLM智能体（其动作是离散文本，必须在观察环境后从策略中采样）场景下不成立。与遭受复合误差的自回归世界模型不同，ADWM将每个转移建模为独立的去噪过程，从而实现可靠的逐步推演，其中世界模型和智能体以因果顺序交替进行。关键在于，被评估的LLM智能体通过策略条件评分函数直接引导每一步的扩散生成，确保模拟轨迹准确反映其决策模式。实验表明，ADWM在多种多轮智能体任务中实现了准确的价值估计和评估可靠性，展示了其作为离线LLM智能体评估实用框架的潜力。

## Abstract
Evaluating large language model (LLM) agents in multi-turn interactive environments is expensive and risky, as it requires online environment interaction. We propose ADWM (Autoregressive Diffusion World Model), an evaluation framework that estimates the performance of a new LLM agent policy purely from pre-collected trajectories. The core idea is to learn a latent diffusion world model that simulates how the environment responds to the evaluation policy, without ever executing it in the real environment. Existing diffusion-based OPE methods guide full trajectories in a single pass by jointly diffusing states and actions, an assumption that breaks down for LLM agents whose actions are discrete text that must be sampled from the policy after observing the environment. Unlike autoregressive world models that suffer from compounding errors, ADWM models each transition as an independent denoising process, enabling reliable step-by-step rollouts where the world model and agent alternate in causal order. Crucially, the LLM agent under evaluation directly guides the diffusion generation at each step via a policy-conditioned score function, ensuring that simulated trajectories accurately reflect its decision-making patterns. Empirically, ADWM achieves accurate value estimates and evaluation reliability across diverse multi-turn agent tasks, demonstrating its promise as a practical framework for offline LLM agent evaluation.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：如何在不与真实环境交互的情况下，仅凭预收集的离线轨迹数据，准确评估一个全新的LLM智能体（agent）策略的性能？
- **背景与动机**：
    - LLM智能体越来越多地部署在多轮交互环境中（如网页导航、代码执行等），其评估通常需要在线执行，成本高（消耗API预算）且有不可逆的风险。
    - 传统离线策略评估（OPE）方法存在局限：
        * 重要性采样（IS）的权重随轨迹长度指数增长，不适合多轮智能体场景。
        * 基于价值的方法（如FQE）在评估策略与行为策略差异大时存在复合偏差。
        * 模型基方法（世界模型）自回归生成观测时，误差会在每步内和跨步间累积（复合误差问题）。
    - 现有扩散模型OPE方法（如Policy-Guided Diffusion）将状态和动作联合扩散，要求动作必须是连续向量且由扩散过程生成，这无法用于LLM智能体（动作是离散文本，必须由LLM在观察到环境后采样）。
- **总体目标**：提出一种新的离线评估框架ADWM，解决上述矛盾，实现从离线数据到新策略价值的可靠估计。

## 2. 论文提出的方法论

### 2.1 核心思想

- **关键洞察**：将全局策略引导的轨迹分布精确分解为一系列单步条件概率的乘积，每个单步条件概率又可以分解为三个因子的乘积（先验、动作后验、策略延续因子）。这种乘积结构正好对应扩散模型中的引导机制，使得被评估的LLM策略可以在每一步去噪过程中通过评分函数梯度进行引导，而无需重新训练世界模型。

### 2.2 关键技术细节

1. **策略引导的轨迹分布**：
   \[
   q_{\alpha}(\tau \mid \pi_e) \propto p_\theta(\tau) \exp\left(\alpha \sum_{t} \log \pi_e(a_t \mid h_t)\right)
   \]
   其中 \(p_\theta\) 是数据集上学习的生成模型，\(\pi_e\) 是评估策略。

2. **单步条件概率分解（定理1）**：
   \[
   \tilde{q}_{\omega,\eta}(o_{t+1} \mid h_t, a_t; \pi_e) \propto \underbrace{P_\theta(o_{t+1} \mid h_t)}_{\text{先验}} \cdot \underbrace{P_\theta(a_t \mid o_{t+1}, h_t)^\omega}_{\text{动作后验}} \cdot \underbrace{C_\alpha(o_{t+1}; h_t, a_t, \pi_e)^\eta}_{\text{策略延续}}
   \]
   - 先验：锚定到离线数据支持。
   - 动作后验：通过无分类器引导（CFG）实现，由超参数 \(\omega\) 控制强度。
   - 策略延续因子：评估未来轨迹与 \(\pi_e\) 的兼容性，通过沿扩散步骤微分 \(\pi_e\) 的对数似然来近似。

3. **引导评分函数（公式9）**：
   \[
   s(z_{t+1}^k, k; h_t^z, a_t, \pi_e) = \nabla \log p_\theta(z_{t+1}^k \mid h_t^z) + \omega \nabla \log p_\phi(a_t \mid z_{t+1}^k, h_t^z) + \eta_k \alpha \nabla_{\hat{z}_{t+1}} \sum_{u=t+1}^T \log \pi_e(\hat{a}_u \mid h_u^\pi)
   \]
   其中第一、二项由CFG实现，第三项通过“去噪后估计+贪婪展开”近似。

4. **架构**（图2）：
   - 编码器 \(E\) 将文本观测映射到潜在空间 \(z_t \in \mathbb{R}^d\)。
   - 投影器 \(G_\psi\) 将潜在映射为软 token \(\tilde{o}_t\)，供评估策略 \(\pi_e\) 读取。
   - 扩散世界模型 \(p_\theta\) 进行 K 步去噪。
   - 奖励头 \(r_\rho\) 预测即时奖励。
   - 终止头判断 episode 是否结束。

5. **训练**：
   - 世界模型通过最大似然学习，损失包括去噪得分匹配、逆动态损失、行为克隆损失、多样性正则化、奖励头、终止头。
   - 所有模块（编码器、世界模型、投影器）端到端联合训练，确保潜在表示由环境动态塑造而非表面语义。

6. **推理（OPE）**：
   - 从行为池中采样初始观测，循环执行：投影→策略采样动作→扩散生成下一个潜在→预测奖励→重复直至终止。
   - 对多轮 rollout 取平均得到价值估计 \(\hat{J}(\pi_e)\)。

## 3. 实验设计

### 3.1 数据集与场景

- 四个LLM智能体基准，覆盖不同奖励结构：
    - **HotpotQA**（密集 F1 奖励）
    - **ScienceWorld**（部分形状奖励）
    - **ALFWorld**（稀疏 0/1 成功奖励）
    - **WebShop**（连续部分奖励）
- 每个基准使用特定的行为策略 \(\pi_b\) 和评估策略 \(\pi_e\)（见表1）：
    - HotpotQA：行为策略为 ReAct-HotpotQA-SFT，评估策略为 DPO 或 PRM 微调版本。
    - ScienceWorld：行为策略为 sw-llama-sft，评估策略为 ETO 优化版。
    - ALFWorld：行为策略为 Llama-3.1-8B-Instr.，评估策略为 LEAP 迭代1和3。
    - WebShop：行为策略为 Llama-3.1-8B-Instr.，评估策略为 LEAP 迭代1。

### 3.2 评估方法

- 为模拟策略质量梯度，对每个评估策略构造 \(\varepsilon\)-greedy 变体（\(\varepsilon \in \{0,0.25,0.5,0.75,1.0\}\)），在真实环境中获取 ground-truth 性能曲线。
- ADWM 在相同 \(\varepsilon\) 下仅通过世界模型 rollout 估计 \(\hat{J}\)，计算 Spearman 秩相关系数。

### 3.3 对比基线

- 五个经典 OPE 方法：
    - 直接方法（DM）
    - 重要性采样（IS）
    - 加权重要性采样（WIS）
    - 拟合 Q 评估（FQE）
    - 双重稳健估计（DR）
- 基线实现遵循 COBS 参考协议。对于需要 \(\pi_b\) 的估计器，提供“特权信息”（精确的每 token 对数概率），这在实际部署中不可用，但作为诊断参考。

## 6. 论文的主要结论与发现

- **ADWM 是唯一在所有 6 个配置上 Spearman \(\rho > 0\) 的估计器**，平均 \(\rho = +0.82\)（最小 \(+0.67\)），而经典基线在多个配置上失败（IS/DR 因重要性权重爆炸，DM 给出固定值，WIS 因权重退化，FQE 除一个案例外均不佳）。
- **消融实验表明三个组件均不可或缺**：局部 CFG 在稀疏奖励任务中关键（WebShop \(\rho\) 从 \(+0.90\) 降至 \(+0.10\)），延续引导在目标导向轨迹中重要（WebShop → \(+0.30\)），\(\psi\) 适配器在语言丰富环境中主导（HotpotQA → \(+0.40\)）。
- **训练收敛稳定**，所有损失在 50 个 epoch 内收敛。
- **世界模型训练仅依赖于行为策略数据**，评估策略 \(\pi_e\) 只在推理时通过梯度参与。

## 7. 优点

1. **理论扎实**：从轨迹级策略引导律推导出精确的单步条件分解，并映射到扩散模型引导机制，有严格的数学证明（定理1，命题1）。
2. **方法新颖**：解决了“扩散生成完整轨迹要求动作先于观测已知”与“LLM动作必须后于观测”之间的因果矛盾，通过每步独立去噪+策略引导实现自回归 rollout。
3. **实用性强**：完全离线训练，无需真实环境交互；单一世界模型可对任意新策略进行评估，无需重新训练。
4. **实验严谨**：覆盖多种奖励结构、多种后训练范式、多种策略对；提供了特权信息给基线作最严格对照；消融实验验证了各组件的必要性。
5. **鲁棒性**：在所有配置上始终保持正相关，优于所有基线。
6. **效率**：世界模型训练仅占总 GPU 小时的 <5%，主要成本来自 LLM 推理，这是任何 OPE 方法都不可避免的。

## 8. 不足与局限

1. **适用范围限制**：
   - \(\psi\) 适配器与特定评估 LLM 的嵌入空间绑定，跨模型家族评估（如用 Llama 训练的世界模型评估 Qwen 策略）需要一个一次性适配器重训练。
   - 像所有 OPE 方法一样，ADWM 受限于 ground-truth 质量：在 WebShop 上，GT 成功率的二项标准误（~7%）与 LEAP 直接迭代间隔（3–19%）接近，使得细粒度区分需要更多 GT episode。
2. **实验覆盖缺口**：
   - 未评估跨模型家族（如 Qwen、GPT 等）的迁移能力。
   - 未在超长 horizon（>30步）或开放域对话等更复杂场景上测试。
   - 消融实验在 WebShop 上每种子方差较大，统计显著性在个别配置（如 ALFWorld-iter1）较低（p=0.219）。
3. **潜在偏差风险**：
   - 世界模型可能过拟合行为数据分布，当 \(\pi_e\) 产生行为分布中罕见动作时，先验因子可能过度约束。
   - 延续引导需要展开未来轨迹的近似，引入近似误差；超参数 \(\alpha, \eta_k\) 的选择可能影响性能。
4. **算力需求**：虽然世界模型训练轻量，但推理阶段需要为每个步骤运行扩散去噪（50步）并多次调用 \(\pi_e\) LLM，在时间和计算上仍有一定开销。
